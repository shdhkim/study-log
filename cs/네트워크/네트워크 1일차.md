# 변조(Modulation)와 반송파(Carrier) — 개념 정리

## 개요

이 문서는 반송파, 변조의 이유, 주요 변조 방식(아날로그·디지털 포함), 그리고 음성/영상 같은 실제 신호를 어떻게 변조·전송하는지를 **쉽고 사례 중심으로** 정리한 Markdown 문서입니다.

---

# 1️⃣ 반송파(Carrier)란?

* **정의**: 정보를 실어 나르는 고주파의 기본 사인파 신호.

* 수식 예시:
  [c(t) = A_c \sin(2\pi f_c t + \phi)]

  * (A_c): 반송파 진폭
  * (f_c): 반송파 주파수
  * (\phi): 위상

* **특징**: 반송파 자체는 의미 있는 정보를 담고 있지 않지만, 정보(저주파 신호)를 실어 보내기 위한 매개체.

---

# 2️⃣ 변조(Modulation)를 하는 이유

1. **장거리 전송**: 저주파(예: 음성 300Hz~3kHz)는 파장이 길어 전송 효율이 낮기 때문에 고주파 반송파에 실어 보냅니다.
2. **안테나 크기 감소**: 파장이 짧아지면 안테나도 짧게 만들 수 있습니다.
3. **다중화(Multiplexing)**: 서로 다른 반송파 주파수를 사용하면 여러 신호를 동시에 보낼 수 있습니다.
4. **잡음 억제**: 고주파 대역은 필터링이 용이해 잡음에 강합니다.

---

# 3️⃣ 기본 관계: 주파수, 파장, 속도

* 기본식: (v = f \lambda)

  * (v): 파동 속도(전자기파의 경우 약 (3×10^8) m/s)
  * (f): 주파수(Hz)
  * (\lambda): 파장(m)

* **속도 일정 시**: (f = v / \lambda) → **파장이 짧을수록 주파수가 높다.**

---

# 4️⃣ 주요 변조 방식

## 아날로그 변조

* **AM (Amplitude Modulation)**: 진폭 변조 (예: AM 라디오)
* **FM (Frequency Modulation)**: 주파수 변조 (예: FM 라디오)
* **PM (Phase Modulation)**: 위상 변조 (위성통신 등)

## 디지털 변조

* **ASK (Amplitude Shift Keying)**: 진폭의 크기로 0과 1 구분.
* **FSK (Frequency Shift Keying)**: 주파수 차이로 0과 1 구분.
* **PSK (Phase Shift Keying)**: 위상 차이로 0과 1 구분.
* **QAM (Quadrature Amplitude Modulation)**: 진폭+위상을 동시에 변조(64QAM, 256QAM 등).

---

# 5️⃣ 왜 변조가 필요한가 (쉽게 이해)

| 이유     | 설명              |
| ------ | --------------- |
| 장거리 전송 | 고주파일수록 멀리 전달 가능 |
| 작은 안테나 | 파장이 짧아져 제작 용이   |
| 다중 전송  | 주파수별 채널 구분 가능   |
| 잡음 감소  | 고주파 대역 필터링 쉬움   |

비유하자면, **저주파 신호는 말소리**고 **고주파 반송파는 라디오 전파**입니다.
사람의 목소리를 직접 보내면 금방 사라지지만, 라디오 전파에 실어 보내면 멀리 도달합니다.

---

# 6️⃣ ASK와 AM 비교

| 구분    | AM        | ASK            |
| ----- | --------- | -------------- |
| 형태    | 아날로그      | 디지털            |
| 변조 요소 | 진폭(연속 변화) | 진폭(0/1 이산 변화)  |
| 예시    | AM 라디오    | OOK, RFID, 광통신 |

---

# 7️⃣ 음성·영상 신호가 전송되는 실제 과정 (쉽게 설명)

## 1. 입력 (아날로그 신호)

* **음성**: 마이크가 공기 진동을 전기 신호로 변환.
* **영상**: 카메라가 빛을 전압 신호로 변환.

## 2. 디지털화 (ADC)

* 소리를 짧은 간격으로 잘라 숫자로 저장 → ‘샘플링’.
* 1초에 44,100번 측정하면 44.1kHz 샘플링.
* 이렇게 얻은 값들은 0과 1로 표현됩니다.

예시:

> “안녕하세요” → 마이크 → 전압 변동 → [0101010010101010...]

---

## 3. 압축 (코덱)

* 용량이 너무 크면 전송이 어렵기 때문에 **압축**합니다.

  * 음성: MP3, AAC, AMR 등.
  * 영상: H.264, HEVC 등.

예시:

> 동영상의 첫 프레임은 전체 화면을, 이후에는 ‘얼굴이 조금 움직인 부분만’ 보냅니다.

---

## 4. 패킷화 및 변조

* 데이터를 작은 조각(패킷)으로 나누어 보냅니다.
* 각 패킷에는 순서 번호와 주소(IP)가 붙습니다.
* 비트(0,1)를 전파로 바꾸는 과정이 **변조(Modulation)**입니다.

| 변조 방식 | 설명          | 예시          |
| ----- | ----------- | ----------- |
| ASK   | 진폭 차이로 표현   | 단거리 무선, 광통신 |
| FSK   | 주파수 차이로 표현  | 블루투스 초기형    |
| PSK   | 위상 차이로 표현   | LTE, 위성통신   |
| QAM   | 진폭+위상 동시 표현 | Wi‑Fi, 5G   |

---

## 5. 전송과 복조

1. **송신 안테나**: 변조된 신호를 전파로 방출.
2. **수신 안테나**: 전파를 받아 전기 신호로 변환.
3. **복조기**: 반송파에서 원래 비트(데이터)를 꺼냄.
4. **디코더**: 압축 해제 후 소리·영상 복원.

결과적으로 사용자는 **“음성 통화”나 “유튜브 영상”**을 시청하게 됩니다.

---

# 8️⃣ 쉬운 비유로 정리

| 단계  | 비유                      |
| --- | ----------------------- |
| 샘플링 | 소리를 초 단위로 잘게 자르는 것      |
| 압축  | 불필요한 부분을 지우는 것          |
| 패킷화 | 편지를 작게 나누어 보내는 것        |
| 변조  | 택배 트럭에 실어 보냄 (전파 위에 얹기) |
| 복조  | 택배를 받아 다시 조립하는 것        |

---

# 9️⃣ 전체 흐름 요약

```
[음성/영상 신호]
   ↓ (아날로그→디지털 변환)
[압축 및 패킷화]
   ↓ (변조)
[전파로 송신]
   ↓
[수신 → 복조 → 복원]
   ↓
[스피커/화면 출력]
```

# 🔟 한 문장 정리

> **변조란, 사람의 목소리나 영상을 0과 1로 바꾸어 반송파(전파)에 실어 보내고, 수신 측에서 복조하여 다시 원래 형태로 되돌리는 기술이다.**
