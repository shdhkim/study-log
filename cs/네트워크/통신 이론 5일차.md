# 네트워크/데이터통신 정리

이 문서는 지금까지 설명했던 내용을 하나의 흐름으로 이해할 수 있도록
정리한 문서이다.

# 1. 데이터 통신이란

데이터 통신은 컴퓨터나 스마트폰이 서로 데이터를 주고받는 과정이다.

예를 들어

- 웹 브라우저 접속
- 카카오톡
- 유튜브
- 인터넷 전화
- 온라인 게임

모두 데이터 통신이다.

모든 통신은 결국

사용자 데이터 → 패킷 생성 → 네트워크 전송 → 상대방 수신

의 과정을 거친다.

---

# 2. LAN, MAN, WAN

## LAN(Local Area Network)

좁은 범위의 네트워크이다.

예

- 집
- 회사
- 학교

같은 LAN에서는 MAC 주소를 이용하여 직접 통신한다.

예

PC ↓

스위치

↓

PC

또는

노트북(Wi-Fi) ↓

AP

↓

스위치

↓

데스크톱

## MAN

도시 규모의 네트워크.

여러 LAN을 연결한다.

예

대학교 캠퍼스 도시 통신망

## WAN

국가, 대륙 규모.

대표적인 예가 인터넷이다.

집 LAN ↓

통신사망

↓

인터넷

↓

회사 LAN

---

# 3. IP와 MAC 주소

## IP

논리 주소.

최종 목적지를 찾는다.

예

192.168.0.10

203.0.113.10

## MAC

랜카드의 주소.

현재 LAN에서 다음 장치에게 전달할 때 사용한다.

예

AA:BB:CC:DD:EE:FF

핵심

IP = 최종 목적지

MAC = 현재 구간의 목적지

---

# 4. Wi-Fi MAC과 Ethernet MAC

MAC은 컴퓨터에 있는 것이 아니라 네트워크 인터페이스에 있다.

노트북

- Ethernet MAC
- Wi-Fi MAC

두 MAC은 일반적으로 서로 다르다.

유선 연결 시에는 Ethernet MAC이 사용되고

무선 연결 시에는 Wi-Fi MAC이 사용된다.

---

# 5. 같은 LAN 통신

예

A

IP 192.168.0.10

MAC AA

B

IP 192.168.0.20

MAC BB

순서

1.  목적지 IP 확인
2.  같은 LAN인지 계산
3.  ARP 수행
4.  B의 MAC 획득
5.  프레임 생성
6.  스위치가 MAC 확인
7.  B에게 전달

라우터는 사용되지 않는다.

---

# 6. ARP

ARP는

IP → MAC

을 찾는 프로토콜이다.

브로드캐스트

FF:FF:FF:FF:FF:FF

를 이용한다.

"192.168.0.20 사용하는 장비 누구?"

라고 묻는다.

상대방이

"내 MAC은 BB..."

라고 응답한다.

응답은 ARP 캐시에 저장된다.

---

# 7. 허브

허브는 가장 단순한 장비이다.

동작

A가 보내면

허브는

모든 포트에 복사한다.

단점

- 충돌
- 성능 저하
- 목적지 구분 불가

---

# 8. 스위치

스위치는 MAC 주소를 학습한다.

MAC 테이블

AA → Port1

BB → Port2

CC → Port3

목적지 MAC을 보고 해당 포트만 전송한다.

같은 LAN 통신의 핵심 장비이다.

---

# 9. 공유기(AP + Switch + Router)

가정용 공유기는 하나의 장비 안에

- AP
- 스위치
- 라우터
- DHCP
- NAT

기능이 함께 들어 있다.

---

# 10. AP의 역할

AP는

무선 LAN과 유선 LAN을 연결하는 2계층 브리지이다.

노트북 ↓

Wi-Fi

↓

AP

↓

스위치

↓

데스크톱

같은 LAN이면 AP는 프레임을 서로 변환하여 전달한다.

---

# 11. 라우터

라우터는 서로 다른 네트워크를 연결한다.

LAN1

192.168.1.0/24

↓

Router

↓

LAN2

192.168.2.0/24

라우터는 IP 주소를 보고 경로를 결정한다.

---

# 12. 라우팅

순서

1.  목적지 IP 확인
2.  라우팅 테이블 조회
3.  다음 홉 결정
4.  새로운 MAC 헤더 생성
5.  다음 네트워크 전송

MAC은 매 홉마다 바뀌고

IP는 기본적으로 유지된다.

---

# 13. 라우팅 테이블

예

192.168.1.0/24 → LAN1

192.168.2.0/24 → LAN2

0.0.0.0/0 → 인터넷

라우터는 가장 적합한 경로를 선택한다.

---

# 14. NAT

사설 IP를 공인 IP로 변환한다.

예

192.168.0.10

↓

1.2.3.4

인터넷에서는 공인 IP가 사용된다.

---

- DNS

# 19. 광통신

전기 대신 빛으로 전송한다.

PC

↓

공유기

↓

ONT

↓

광케이블

↓

통신사

빛은 전기보다

- 손실이 적고
- 속도가 빠르고
- 먼 거리 전송이 가능하다.

---

# 20. OSI 7계층 요약

1 물리계층 케이블, 광케이블

2 데이터링크 MAC, Ethernet, Wi-Fi, Switch

3 네트워크 IP, Router

4 전송 TCP, UDP

5 세션 연결 관리

6 표현 암호화(TLS), 압축

7 응용 HTTP, FTP, SMTP, DNS

---

# 21. 핵심 비교

허브 - 모두에게 전송 - 주소 사용 안 함

스위치 - MAC 주소 - 같은 LAN

라우터 - IP 주소 - 다른 LAN

ARP - IP → MAC

DNS - 도메인 → IP

DHCP - IP 자동 할당

NAT - 사설 ↔ 공인 IP

AP - Wi-Fi와 Ethernet 연결

---

# 최종 핵심

같은 LAN

IP 확인 → ARP → MAC 획득 → 스위치 → 상대 PC

다른 LAN

IP 확인 → 게이트웨이 → 라우터 → 라우팅 → 다음 홉 → 목적지

가장 중요한 개념

IP는 최종 목적지를 찾는다.

MAC은 현재 구간에서 다음 장비를 찾는다.

스위치는 MAC을 사용한다.

라우터는 IP를 사용한다.

AP는 Wi-Fi와 Ethernet을 하나의 LAN으로 연결한다.

허브는 모든 포트로 데이터를 복사한다.
