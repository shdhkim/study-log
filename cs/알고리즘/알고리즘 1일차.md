# 2. 기본 Java 템플릿

알고리즘 문제를 풀 때 가장 많이 쓰는 기본 구조입니다.

```java
import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringBuilder sb = new StringBuilder();

        int n = Integer.parseInt(br.readLine());

        sb.append(n);

        System.out.println(sb);
    }
}
```

## 코드 설명

```java
import java.io.*;
```

입출력 관련 클래스를 사용하기 위한 import입니다.

대표적으로 다음 클래스들을 사용합니다.

- `BufferedReader`
- `InputStreamReader`
- `IOException`

```java
import java.util.*;
```

자료구조와 유틸 클래스를 사용하기 위한 import입니다.

대표적으로 다음 클래스들을 사용합니다.

- `Arrays`
- `ArrayList`
- `HashMap`
- `HashSet`
- `Queue`
- `Deque`
- `PriorityQueue`
- `StringTokenizer`

```java
public class Main
```

대부분의 온라인 저지에서는 클래스 이름을 `Main`으로 요구합니다.

```java
public static void main(String[] args) throws Exception
```

프로그램의 시작점입니다.

`throws Exception`을 붙이면 `BufferedReader` 사용 시 발생할 수 있는 예외 처리를 간단히 넘길 수 있습니다.

# 3. 입력 처리

## 3.1 Scanner보다 BufferedReader를 많이 쓰는 이유

`Scanner`는 사용하기 쉽지만 입력이 많으면 느릴 수 있습니다.

```java
Scanner sc = new Scanner(System.in);
int n = sc.nextInt();
```

간단한 문제에서는 괜찮지만, 입력이 수만 줄 이상이면 `BufferedReader`가 더 안전합니다.

```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
```

## 3.2 BufferedReader 기본 사용법

```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

String line = br.readLine();
int n = Integer.parseInt(line);
```

## 코드 설명

```java
br.readLine()
```

한 줄을 문자열로 읽습니다.

예를 들어 입력이 다음과 같다면:

```text
123
```

`br.readLine()`의 결과는 `"123"`입니다.

숫자 `123`이 아니라 문자열 `"123"`이므로 정수로 바꾸려면 다음처럼 해야 합니다.

```java
int n = Integer.parseInt(line);
```

# 4. 공백 기준 입력: StringTokenizer

입력이 한 줄에 여러 개 들어오는 경우가 많습니다.

```text
10 20
```

이 경우 다음처럼 처리합니다.

```java
StringTokenizer st = new StringTokenizer(br.readLine());

int a = Integer.parseInt(st.nextToken());
int b = Integer.parseInt(st.nextToken());
```

## 코드 설명

```java
new StringTokenizer(br.readLine())
```

한 줄을 읽고 공백 기준으로 잘라줍니다.

```java
st.nextToken()
```

다음 토큰을 하나 꺼냅니다.

입력이 `"10 20"`이면:

1. 첫 번째 `nextToken()`은 `"10"`
2. 두 번째 `nextToken()`은 `"20"`

입니다.

# 5. 여러 줄 입력 처리

## 예시 입력

```text
5
1 2 3 4 5
```

## 코드

```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

int n = Integer.parseInt(br.readLine());

int[] arr = new int[n];

StringTokenizer st = new StringTokenizer(br.readLine());

for (int i = 0; i < n; i++) {
    arr[i] = Integer.parseInt(st.nextToken());
}
```

## 코드 설명

```java
int n = Integer.parseInt(br.readLine());
```

첫 줄에서 배열 크기를 받습니다.

```java
int[] arr = new int[n];
```

크기가 `n`인 정수 배열을 만듭니다.

```java
StringTokenizer st = new StringTokenizer(br.readLine());
```

두 번째 줄 `"1 2 3 4 5"`를 공백 기준으로 분리합니다.

```java
for (int i = 0; i < n; i++)
```

배열 크기만큼 반복하면서 값을 넣습니다.

# 6. 출력 처리: StringBuilder

출력이 많을 때는 매번 `System.out.println()`을 쓰는 것보다 `StringBuilder`에 모아서 한 번에 출력하는 것이 좋습니다.

## 비효율적인 방식

```java
for (int i = 0; i < 100000; i++) {
    System.out.println(i);
}
```

출력 횟수가 많으면 느려질 수 있습니다.

## 권장 방식

```java
StringBuilder sb = new StringBuilder();

for (int i = 0; i < 100000; i++) {
    sb.append(i).append('\n');
}

System.out.print(sb);
```

## 코드 설명

```java
sb.append(i)
```

숫자 `i`를 문자열처럼 뒤에 붙입니다.

```java
.append('\n')
```

줄바꿈을 추가합니다.

```java
System.out.print(sb)
```

마지막에 한 번만 출력합니다.

# 7. 배열 기본

## 배열 생성

```java
int[] arr = new int[5];
```

크기가 5인 정수 배열입니다.

기본값은 모두 0입니다.

처음부터 값을 넣어서 만들 수도 있습니다.

## 배열 접근

```java

```

배열은 0번 인덱스부터 시작합니다.


```java
for (int i = 0; i < arr.length; i++) {
```

## 코드 설명

```java
arr.length
```

배열의 길이입니다.

문자열은 `length()`를 쓰지만 배열은 `length`입니다.

```java
s.length();  // 문자열
arr.length;  // 배열
```

# 8. 향상된 for문

```java
int[] arr = {1, 2, 3};

for (int x : arr) {
    System.out.println(x);
}
```

## 의미

배열의 값을 하나씩 꺼내서 `x`에 넣습니다.

위 코드는 다음 코드와 비슷합니다.

```java
for (int i = 0; i < arr.length; i++) {
    int x = arr[i];
    System.out.println(x);
}
```

## 주의점

향상된 for문은 인덱스가 필요 없는 경우에 좋습니다.

하지만 인덱스가 필요하면 일반 for문을 써야 합니다.

```java
for (int i = 0; i < arr.length; i++) {
    System.out.println(i + " " + arr[i]);
}
```

# 12. List에서 remove 주의

`List<Integer>`에서 `remove(1)`은 값 1을 지우는 것이 아니라 1번 인덱스를 지웁니다.

```java
List<Integer> list = new ArrayList<>();
list.add(10);
list.add(20);
list.add(30);

list.remove(1);

System.out.println(list);
```

출력:

```text
[10, 30]
```

값 `1`이 아니라 인덱스 `1`의 값인 `20`이 삭제됩니다.

## 값으로 삭제하고 싶다면

```java
list.remove(Integer.valueOf(20));
```

# 15. getOrDefault

빈도수 계산에서 매우 자주 사용합니다.

```java
Map<String, Integer> map = new HashMap<>();

String[] arr = {"A", "B", "A", "C", "A"};

for (String s : arr) {
    map.put(s, map.getOrDefault(s, 0) + 1);
}

System.out.println(map);
```

## 코드 설명

```java
map.getOrDefault(s, 0)
```

키 `s`가 있으면 기존 값을 가져옵니다.

키 `s`가 없으면 기본값 0을 사용합니다.

따라서:

```java
map.put(s, map.getOrDefault(s, 0) + 1);
```

은 다음 의미입니다.

```text
기존 개수 가져오기
없으면 0
거기에 1 더하기
다시 저장
```

# 16. Queue

`Queue`는 먼저 들어간 값이 먼저 나오는 FIFO 구조입니다.

```java
Queue<Integer> q = new LinkedList<>();

q.offer(10);
q.offer(20);

System.out.println(q.poll());
System.out.println(q.poll());
```

출력:

```text
10
20
```

## 주요 메서드

| 메서드    | 의미          |
| --------- | ------------- |
| offer(x)  | 값 추가       |
| poll()    | 값 꺼내기     |
| peek()    | 맨 앞 값 확인 |
| isEmpty() | 비었는지 확인 |

## 언제 사용하나

- BFS
- 대기열
- 순서대로 처리하는 문제

# 18. PriorityQueue

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
```

출력:


```java

pq.offer(2);

System.out.println(pq.poll());
```

출력:

```text
3
```

# 22. Comparator

객체나 2차원 배열을 원하는 기준으로 정렬할 때 사용합니다.

## 2차원 배열 정렬

```java
int[][] arr = {
    {1, 5},
    {2, 3},
    {3, 4}
};

Arrays.sort(arr, (a, b) -> Integer.compare(a[1], b[1]));

for (int[] x : arr) {
    System.out.println(Arrays.toString(x));
}
```

## 코드 설명

```java
(a, b) -> Integer.compare(a[1], b[1])
```

두 배열 `a`, `b`를 비교합니다.

`a[1]`과 `b[1]`을 기준으로 오름차순 정렬합니다.

## 주의

다음 방식은 오버플로우 위험이 있습니다.

```java
Arrays.sort(arr, (a, b) -> a[1] - b[1]);
```

안전하게는 다음처럼 씁니다.

```java
Arrays.sort(arr, (a, b) -> Integer.compare(a[1], b[1]));
```


# 24. 시간복잡도 기본

## O(1)

```java
int x = arr[0];
```

배열 인덱스 접근은 한 번에 가능하므로 O(1)입니다.

## O(N)

```java
for (int i = 0; i < n; i++) {
    System.out.println(i);
}
```

n번 반복하므로 O(N)입니다.

## O(N^2)

```java
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        System.out.println(i + " " + j);
    }
}
```

n번 반복 안에서 다시 n번 반복하므로 O(N^2)입니다.

## O(log N)

이분탐색처럼 매번 탐색 범위가 절반으로 줄어드는 경우입니다.

## O(N log N)

정렬에서 자주 나오는 시간복잡도입니다.

# 25. 자주 쓰는 코드 패턴 정리


## Set 중복 체크

```java
Set<Integer> set = new HashSet<>();

for (int x : arr) {
    if (set.contains(x)) {
        System.out.println("중복 발견");
    }

    set.add(x);
}
```

# 26. 자주 틀리는 부분

## 1. 배열 범위 초과

```java
for (int i = 0; i <= arr.length; i++) {
    System.out.println(arr[i]);
}
```

위 코드는 틀립니다.

배열 마지막 인덱스는 `arr.length - 1`입니다.

정답:

```java
for (int i = 0; i < arr.length; i++) {
    System.out.println(arr[i]);
}
```

## 2. 문자열 비교

```java
if (s == "hello") {
}
```

문자열 내용 비교는 `equals()`를 사용해야 합니다.

```java
if (s.equals("hello")) {
}
```

## 3. Integer remove 혼동

```java
list.remove(1);
```

`List<Integer>`에서 이 코드는 값 1 삭제가 아니라 인덱스 1 삭제입니다.

값을 삭제하려면:

```java
list.remove(Integer.valueOf(1));
```

## 4. int 오버플로우

```java
int sum = 0;

for (int x : arr) {
    sum += x;
}
```

합이 커질 수 있으면:

```java
long sum = 0;

for (int x : arr) {
    sum += x;
}
```
