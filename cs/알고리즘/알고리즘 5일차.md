# 2. 재귀란?

재귀는 함수가 자기 자신을 다시 호출하는 방식입니다.

예를 들어 다음 코드는 자기 자신을 호출합니다.

```java
public static void func() {
    func();
}
```

하지만 위 코드는 종료 조건이 없기 때문에 무한 호출됩니다.

재귀에는 반드시 종료 조건이 있어야 합니다.

---

# 3. 재귀의 기본 구조

```java
public static void recursive(int depth) {
    if (depth == 3) {
        return;
    }

    System.out.println(depth);

    recursive(depth + 1);
}
```

## 전체 코드

```java
public class Main {
    public static void recursive(int depth) {
        if (depth == 3) {
            return;
        }

        System.out.println(depth);

        recursive(depth + 1);
    }

    public static void main(String[] args) {
        recursive(0);
    }
}
```

## 출력

```text
0
1
2
```

## 코드 설명

```java
recursive(0);
```

depth 0부터 시작합니다.

```java
if (depth == 3) {
    return;
}
```

depth가 3이면 재귀를 멈춥니다.

이 부분이 종료 조건입니다.

```java
System.out.println(depth);
```

현재 depth를 출력합니다.

```java
recursive(depth + 1);
```

depth를 1 증가시켜 자기 자신을 다시 호출합니다.

호출 흐름:

```text
recursive(0)
 -> recursive(1)
   -> recursive(2)
     -> recursive(3)
        return
```

# 4. 재귀에서 중요한 2가지

재귀를 만들 때는 항상 다음 두 가지를 먼저 생각해야 합니다.

## 1. 종료 조건

언제 멈출 것인가?

```java
if (depth == n) {
    return;
}
```

## 2. 다음 호출

다음 단계로 어떻게 이동할 것인가?

```java
recursive(depth + 1);
```

종료 조건이 없으면 무한 재귀가 됩니다.

다음 호출에서 상태가 변하지 않으면 역시 무한 재귀가 됩니다.

---

# 5. 재귀 호출 순서 이해

```java
public static void recursive(int depth) {
    if (depth == 3) {
        return;
    }

    System.out.println("before " + depth);

    recursive(depth + 1);

    System.out.println("after " + depth);
}
```

## 출력

```text
before 0
before 1
before 2
after 2
after 1
after 0
```

## 코드 설명

재귀 호출 전 코드는 내려가면서 실행됩니다.

```java
System.out.println("before " + depth);
```

재귀 호출 후 코드는 재귀가 끝나고 돌아오면서 실행됩니다.

```java
System.out.println("after " + depth);
```

호출 흐름:

```text
recursive(0)
 before 0
 recursive(1)
   before 1
   recursive(2)
     before 2
     recursive(3)
     return
     after 2
   after 1
 after 0
```

# 6. 팩토리얼

팩토리얼은 재귀의 대표 예제입니다.

```text
5! = 5 * 4 * 3 * 2 * 1
```

## 코드

```java
public static int factorial(int n) {
    if (n == 1) {
        return 1;
    }

    return n * factorial(n - 1);
}
```

## 전체 코드

```java
public class Main {
    public static int factorial(int n) {
        if (n == 1) {
            return 1;
        }

        return n * factorial(n - 1);
    }

    public static void main(String[] args) {
        System.out.println(factorial(5));
    }
}
```

## 출력

```text
120
```

## 코드 설명

```java
if (n == 1) {
    return 1;
}
```

1!은 1이므로 종료 조건입니다.

```java
return n * factorial(n - 1);
```

n!은 n과 (n-1)!의 곱입니다.

호출 흐름:

```text
factorial(5)
= 5 * factorial(4)
= 5 * 4 * factorial(3)
= 5 * 4 * 3 * factorial(2)
= 5 * 4 * 3 * 2 * factorial(1)
= 5 * 4 * 3 * 2 * 1
= 120
```

# 7. 피보나치 재귀

피보나치 수열은 다음과 같습니다.

```text
0, 1, 1, 2, 3, 5, 8, 13 ...
```

점화식:

```text
f(n) = f(n - 1) + f(n - 2)
```

## 코드

```java
public static int fib(int n) {
    if (n == 0) return 0;
    if (n == 1) return 1;

    return fib(n - 1) + fib(n - 2);
}
```

## 주의

이 방식은 중복 계산이 많아서 느립니다.

예를 들어 `fib(5)`를 구할 때 `fib(3)`, `fib(2)` 같은 값이 여러 번 계산됩니다.

시간복잡도는 대략 O(2^N)에 가깝습니다.

피보나치는 DP에서 다시 더 효율적으로 다루는 것이 좋습니다.

# 8. 완전탐색이란?

완전탐색은 가능한 모든 경우를 전부 확인하는 방식입니다.

예를 들어 배열에서 두 수의 합이 target인지 확인한다고 합시다.

가장 단순한 방법은 모든 쌍을 확인하는 것입니다.

```java
int[] arr = {1, 2, 3, 4};
int target = 5;

for (int i = 0; i < arr.length; i++) {
    for (int j = i + 1; j < arr.length; j++) {
        if (arr[i] + arr[j] == target) {
            System.out.println(arr[i] + " " + arr[j]);
        }
    }
}
```

## 코드 설명

```java
for (int i = 0; i < arr.length; i++)
```

첫 번째 숫자를 고릅니다.

```java
for (int j = i + 1; j < arr.length; j++)
```

두 번째 숫자를 고릅니다.

`j = i + 1`로 시작하는 이유는 같은 원소를 두 번 고르지 않고, 중복 쌍도 피하기 위해서입니다.

시간복잡도는 O(N^2)입니다.

# 11. DFS형 재귀

DFS는 Depth First Search, 깊이 우선 탐색입니다.

재귀로 많이 구현합니다.

가장 기본적인 형태는 다음과 같습니다.

```java
public static void dfs(int depth) {
    if (depth == 목표깊이) {
        return;
    }

    for (가능한 선택들) {
        선택;
        dfs(depth + 1);
        선택 취소;
    }
}
```

이 구조가 백트래킹의 기본 구조입니다.

# 12. 백트래킹이란?

백트래킹은 모든 경우를 탐색하되, 가능성이 없는 경우는 더 이상 탐색하지 않고 되돌아가는 방식입니다.

핵심은 다음 두 가지입니다.

- 선택한다
- 재귀 호출한다
- 선택을 취소한다

구조:

```java
choose
dfs
unchoose
```

Java 코드로 표현하면:

```java
visited[i] = true;
dfs(depth + 1);
visited[i] = false;
```

# 13. 순열

순열은 순서가 중요한 선택입니다.

예를 들어 1, 2, 3 중 2개를 고르는 순열은 다음과 같습니다.

```text
1 2
1 3
2 1
2 3
3 1
3 2
```

`1 2`와 `2 1`은 다릅니다.

---

# 14. 순열 코드

```java
import java.util.*;

public class Main {
    static int n = 3;
    static int r = 2;
    static int[] arr = {1, 2, 3};
    static int[] result = new int[r];
    static boolean[] visited = new boolean[n];

    public static void permutation(int depth) {
        if (depth == r) {
            System.out.println(Arrays.toString(result));
            return;
        }

        for (int i = 0; i < n; i++) {
            if (!visited[i]) {
                visited[i] = true;
                result[depth] = arr[i];

                permutation(depth + 1);

                visited[i] = false;
            }
        }
    }

    public static void main(String[] args) {
        permutation(0);
    }
}
```

## 출력

```text
[1, 2]
[1, 3]
[2, 1]
[2, 3]
[3, 1]
[3, 2]
```

## 코드 설명

```java
static int n = 3;
static int r = 2;
```

3개 중에서 2개를 뽑습니다.

```java
static int[] result = new int[r];
```

현재 선택한 결과를 저장합니다.

```java
static boolean[] visited = new boolean[n];
```

이미 선택한 원소를 다시 선택하지 않기 위해 사용합니다.

```java
if (depth == r)
```

r개를 모두 선택했다면 하나의 순열이 완성된 것입니다.

```java
for (int i = 0; i < n; i++)
```

모든 원소를 후보로 확인합니다.

```java
if (!visited[i])
```

아직 선택하지 않은 원소만 선택합니다.

```java
visited[i] = true;
result[depth] = arr[i];
```

현재 원소를 선택합니다.

```java
permutation(depth + 1);
```

다음 자리를 선택하러 갑니다.

```java
visited[i] = false;
```

재귀가 끝나고 돌아오면 선택을 취소합니다.

이 취소 과정이 없으면 다른 경우를 만들 수 없습니다.

---

# 15. 순열에서 visited가 필요한 이유

순열은 같은 원소를 중복해서 쓰면 안 됩니다.

예를 들어 1, 2, 3 중 2개를 뽑을 때:

```text
1 1
2 2
3 3
```

이런 결과는 나오면 안 됩니다.

따라서 이미 사용한 원소는 `visited`로 막습니다.

# 16. 조합

조합은 순서가 중요하지 않은 선택입니다.

1, 2, 3 중 2개를 고르는 조합은 다음과 같습니다.

```text
1 2
1 3
2 3
```

`1 2`와 `2 1`은 같은 것으로 봅니다.

---

# 17. 조합 코드

```java
import java.util.*;

public class Main {
    static int n = 3;
    static int r = 2;
    static int[] arr = {1, 2, 3};
    static int[] result = new int[r];

    public static void combination(int start, int depth) {
        if (depth == r) {
            System.out.println(Arrays.toString(result));
            return;
        }

        for (int i = start; i < n; i++) {
            result[depth] = arr[i];
            combination(i + 1, depth + 1);
        }
    }

    public static void main(String[] args) {
        combination(0, 0);
    }
}
```

## 출력

```text
[1, 2]
[1, 3]
[2, 3]
```

## 코드 설명

```java
combination(int start, int depth)
```

`start`는 이번 선택에서 시작할 인덱스입니다.

`depth`는 현재 몇 개를 선택했는지 나타냅니다.

```java
for (int i = start; i < n; i++)
```

start부터 뒤쪽 원소만 선택합니다.

이렇게 해야 중복 조합이 나오지 않습니다.

```java
result[depth] = arr[i];
```

현재 depth 위치에 값을 저장합니다.

```java
combination(i + 1, depth + 1);
```

다음 선택은 현재 i보다 뒤에서 시작합니다.

그래서 `2 1` 같은 역순 조합이 나오지 않습니다.

# 18. 순열과 조합 차이

| 구분         | 순열           | 조합           |

| 순서         | 중요           | 중요하지 않음  |

# 19. 부분집합

부분집합은 각 원소를 선택하거나 선택하지 않는 모든 경우입니다.

예를 들어 {1, 2, 3}의 부분집합은 다음과 같습니다.

```text
{}
{1}
{2}
{3}
{1,2}
{1,3}
{2,3}
{1,2,3}
```

원소가 N개면 부분집합 개수는 2^N개입니다.

---

# 20. 부분집합 코드

```java
import java.util.*;

public class Main {
    static int[] arr = {1, 2, 3};
    static boolean[] selected = new boolean[arr.length];

    public static void subset(int depth) {
        if (depth == arr.length) {
            for (int i = 0; i < arr.length; i++) {
                if (selected[i]) {
                    System.out.print(arr[i] + " ");
                }
            }
            System.out.println();
            return;
        }

        selected[depth] = true;
        subset(depth + 1);

        selected[depth] = false;
        subset(depth + 1);
    }

    public static void main(String[] args) {
        subset(0);
    }
}
```

## 코드 설명

```java
selected[depth] = true;
subset(depth + 1);
```

현재 원소를 선택하는 경우입니다.

```java
selected[depth] = false;
subset(depth + 1);
```

현재 원소를 선택하지 않는 경우입니다.

각 원소마다 선택/미선택 2가지 경우가 있으므로 전체 경우의 수는 2^N입니다.

# 21. 백트래킹에서 가지치기

가지치기는 더 이상 볼 필요 없는 경우를 중간에 끊는 것입니다.

예를 들어 합이 target을 넘으면 더 탐색하지 않는 경우입니다.

```java
public static void dfs(int depth, int sum) {
    if (sum > target) {
        return;
    }

    if (depth == n) {
        if (sum == target) {
            count++;
        }
        return;
    }

    dfs(depth + 1, sum + arr[depth]);
    dfs(depth + 1, sum);
}
```

## 코드 설명

```java
if (sum > target) {
    return;
}
```

현재 합이 이미 target을 넘었다면 더 진행해도 정답이 될 수 없다고 판단하고 중단합니다.

이런 방식으로 탐색 횟수를 줄일 수 있습니다.

## 주의

이 가지치기는 배열 값이 모두 양수일 때 안전합니다.

음수가 있으면 나중에 합이 다시 줄어들 수 있으므로 단순히 `sum > target`으로 끊으면 안 될 수 있습니다.

# 22. 부분집합 합 문제

배열에서 일부 원소를 골라 target을 만들 수 있는지 확인하는 문제입니다.

```java
public class Main {
    static int[] arr = {1, 2, 3, 4};
    static int target = 6;
    static boolean found = false;

    public static void dfs(int depth, int sum) {
        if (depth == arr.length) {
            if (sum == target) {
                found = true;
            }
            return;
        }

        dfs(depth + 1, sum + arr[depth]);
        dfs(depth + 1, sum);
    }

    public static void main(String[] args) {
        dfs(0, 0);
        System.out.println(found);
    }
}
```

## 코드 설명

```java
dfs(depth + 1, sum + arr[depth]);
```

현재 원소를 선택하는 경우입니다.

```java
dfs(depth + 1, sum);
```

현재 원소를 선택하지 않는 경우입니다.

```java
if (depth == arr.length)
```

모든 원소에 대해 선택 여부를 결정한 상태입니다.

# 23. N개 중 M개 고르기 문제 패턴

알고리즘 문제에서 자주 나오는 형태입니다.

## 순열형

```text
N개 중 M개를 고르는데 순서가 중요하다.
```

사용:

```java
visited
depth
```

## 조합형

```text
N개 중 M개를 고르는데 순서가 중요하지 않다.
```

사용:

```java
start
depth
```

## 부분집합형

```text
각 원소를 고르거나 안 고른다.
```

사용:

```java
selected
depth
```

# 24. 중복 순열

중복 순열은 같은 원소를 여러 번 선택할 수 있습니다.

예를 들어 1, 2, 3 중 2개를 뽑되 중복을 허용하면:

```text
1 1
1 2
1 3
2 1
2 2
2 3
3 1
3 2
3 3
```

## 코드

```java
import java.util.*;

public class Main {
    static int[] arr = {1, 2, 3};
    static int r = 2;
    static int[] result = new int[r];

    public static void repeatedPermutation(int depth) {
        if (depth == r) {
            System.out.println(Arrays.toString(result));
            return;
        }

        for (int i = 0; i < arr.length; i++) {
            result[depth] = arr[i];
            repeatedPermutation(depth + 1);
        }
    }

    public static void main(String[] args) {
        repeatedPermutation(0);
    }
}
```

## 코드 설명

중복을 허용하므로 `visited`가 필요 없습니다.

모든 depth에서 전체 원소를 다시 후보로 사용합니다.

# 25. 중복 조합

중복 조합은 같은 원소를 여러 번 선택할 수 있지만 순서는 중요하지 않습니다.

예를 들어 1, 2, 3 중 2개를 중복 허용 조합으로 고르면:

```text
1 1
1 2
1 3
2 2
2 3
3 3
```

## 코드

```java
import java.util.*;

public class Main {
    static int[] arr = {1, 2, 3};
    static int r = 2;
    static int[] result = new int[r];

    public static void repeatedCombination(int start, int depth) {
        if (depth == r) {
            System.out.println(Arrays.toString(result));
            return;
        }

        for (int i = start; i < arr.length; i++) {
            result[depth] = arr[i];
            repeatedCombination(i, depth + 1);
        }
    }

    public static void main(String[] args) {
        repeatedCombination(0, 0);
    }
}
```

## 코드 설명

```java
repeatedCombination(i, depth + 1);
```

중복을 허용하기 때문에 다음 시작 인덱스를 `i + 1`이 아니라 `i`로 둡니다.

일반 조합은:

```java
combination(i + 1, depth + 1);
```

중복 조합은:

```java
repeatedCombination(i, depth + 1);
```

# 26. DFS로 격자 탐색

2차원 배열에서 상하좌우로 이동하는 문제도 재귀 DFS로 풀 수 있습니다.

## 방향 배열

```java
static int[] dx = {-1, 1, 0, 0};
static int[] dy = {0, 0, -1, 1};
```

의미:

```text
상: (-1, 0)
하: (1, 0)
좌: (0, -1)
우: (0, 1)
```

---

# 27. 격자 DFS 코드

```java
public class Main {
    static int n = 3;
    static int m = 3;
    static int[][] board = {
        {1, 1, 0},
        {0, 1, 0},
        {1, 1, 1}
    };
    static boolean[][] visited = new boolean[n][m];

    static int[] dx = {-1, 1, 0, 0};
    static int[] dy = {0, 0, -1, 1};

    public static void dfs(int x, int y) {
        visited[x][y] = true;

        for (int dir = 0; dir < 4; dir++) {
            int nx = x + dx[dir];
            int ny = y + dy[dir];

            if (nx < 0 || ny < 0 || nx >= n || ny >= m) {
                continue;
            }

            if (visited[nx][ny]) {
                continue;
            }

            if (board[nx][ny] == 0) {
                continue;
            }

            dfs(nx, ny);
        }
    }

    public static void main(String[] args) {
        dfs(0, 0);
    }
}
```

## 코드 설명

```java
visited[x][y] = true;
```

현재 위치를 방문 처리합니다.

```java
for (int dir = 0; dir < 4; dir++)
```

상하좌우 4방향을 확인합니다.

```java
int nx = x + dx[dir];
int ny = y + dy[dir];
```

다음 위치를 계산합니다.

```java
if (nx < 0 || ny < 0 || nx >= n || ny >= m)
```

격자 밖으로 나가면 무시합니다.

```java
if (visited[nx][ny])
```

이미 방문한 곳이면 무시합니다.

```java
if (board[nx][ny] == 0)
```

이동할 수 없는 칸이면 무시합니다.

```java
dfs(nx, ny);
```

이동 가능한 칸이면 재귀로 탐색합니다.

# 28. N-Queen 기초

N-Queen은 N x N 체스판에 퀸 N개를 서로 공격하지 않게 배치하는 문제입니다.

퀸은 다음 방향으로 공격할 수 있습니다.

- 같은 행
- 같은 열
- 대각선

보통 한 행에 퀸 하나씩 놓는 방식으로 풉니다.

---

# 29. N-Queen 코드

```java
public class Main {
    static int n = 4;
    static int[] col = new int[n];
    static int count = 0;

    public static void nQueen(int row) {
        if (row == n) {
            count++;
            return;
        }

        for (int c = 0; c < n; c++) {
            col[row] = c;

            if (isPossible(row)) {
                nQueen(row + 1);
            }
        }
    }

    public static boolean isPossible(int row) {
        for (int i = 0; i < row; i++) {
            if (col[i] == col[row]) {
                return false;
            }

            if (Math.abs(row - i) == Math.abs(col[row] - col[i])) {
                return false;
            }
        }

        return true;
    }

    public static void main(String[] args) {
        nQueen(0);
        System.out.println(count);
    }
}
```

## 코드 설명

```java
static int[] col = new int[n];
```

`col[row] = c`는 row행에 있는 퀸이 c열에 놓였다는 뜻입니다.

```java
nQueen(row)
```

row번째 행에 퀸을 놓는 함수입니다.

```java
if (row == n)
```

0행부터 n-1행까지 모두 퀸을 놓았다면 하나의 정답입니다.

```java
for (int c = 0; c < n; c++)
```

현재 row에서 가능한 모든 열을 시도합니다.

```java
col[row] = c;
```

현재 행 row에 퀸을 c열에 둡니다.

```java
if (isPossible(row))
```

지금까지 배치한 퀸들과 충돌하지 않는지 확인합니다.

```java
nQueen(row + 1);
```

가능하면 다음 행으로 넘어갑니다.

## 대각선 검사

```java
Math.abs(row - i) == Math.abs(col[row] - col[i])
```

행 차이와 열 차이가 같으면 대각선에 있는 것입니다.

# 30. 백트래킹에서 전역 변수 사용

순열/조합/DFS 문제에서는 전역 변수를 자주 씁니다.

예:

```java
static int n;
static int[] arr;
static boolean[] visited;
static int answer;
```

## 왜 전역 변수를 쓰나

재귀 함수마다 여러 값을 계속 넘기면 코드가 복잡해질 수 있습니다.

그래서 공통으로 사용하는 배열, 방문 여부, 정답 변수는 static으로 두는 경우가 많습니다.

하지만 너무 많은 전역 변수는 코드 이해를 어렵게 만들 수 있으므로 필요한 것만 두는 것이 좋습니다.

# 31. 재귀와 스택오버플로우

재귀 호출이 너무 깊으면 StackOverflowError가 발생할 수 있습니다.

예:

```java
public static void dfs(int x) {
    dfs(x + 1);
}
```

종료 조건이 없으므로 무한 호출됩니다.

또는 종료 조건이 있어도 호출 깊이가 너무 크면 문제가 될 수 있습니다.

입력 크기가 매우 큰 그래프 탐색에서는 재귀 대신 반복문 + Stack/Queue를 고려해야 합니다.

---

# 32. 시간복잡도 정리

| 주제              |                     시간복잡도 |
| ----------------- | -----------------------------: |
| 재귀 1회 호출 N번 |                           O(N) |
| 이중 완전탐색     |                         O(N^2) |
| 순열 N개 전체     |                          O(N!) |
| N개 중 R개 순열   |                         O(NPr) |
| 조합              |                         O(NCr) |
| 부분집합          |                         O(2^N) |
| 격자 DFS          |                      O(N \* M) |
| N-Queen           | 매우 큼, 백트래킹으로 가지치기 |

# 33. 자주 틀리는 부분

## 1. 종료 조건 누락

```java
public static void dfs(int depth) {
    dfs(depth + 1);
}
```

종료 조건이 없으므로 무한 재귀입니다.

정상 코드:

```java
public static void dfs(int depth) {
    if (depth == n) {
        return;
    }

    dfs(depth + 1);
}
```

## 2. visited 원복 누락

```java
visited[i] = true;
dfs(depth + 1);
```

이렇게만 쓰면 다른 경우에서 i를 다시 사용할 수 없습니다.

정상 코드:

```java
visited[i] = true;
dfs(depth + 1);
visited[i] = false;
```

## 3. 조합에서 start 실수

조합은 다음 호출에서 `i + 1`을 넘겨야 중복 선택을 막을 수 있습니다.

```java
combination(i + 1, depth + 1);
```

## 4. 순열과 조합 혼동

순서가 중요하면 순열입니다.

순서가 중요하지 않으면 조합입니다.

## 5. 부분집합에서 경우 두 개를 모두 호출하지 않음

부분집합은 선택하는 경우와 선택하지 않는 경우가 모두 필요합니다.

```java
subset(depth + 1); // 선택
subset(depth + 1); // 미선택
```

실제 코드는 선택 상태를 다르게 해야 합니다.

## 6. DFS 격자 범위 체크 누락

```java
if (nx < 0 || ny < 0 || nx >= n || ny >= m) {
    continue;
}
```

이 조건이 없으면 배열 범위 초과가 발생합니다.

# 34. 실전 선택 기준

## 재귀를 의심할 때

- 자기 자신과 비슷한 작은 문제로 나눌 수 있다
- 깊이가 존재한다
- 선택을 반복한다
- 트리 구조처럼 뻗어나간다

## 완전탐색을 의심할 때

- 입력 크기가 작다
- 모든 경우를 확인해야 한다
- 조건이 복잡해서 단순 공식이 없다

## 백트래킹을 의심할 때

- 모든 경우를 탐색하되 중간에 막히는 경우가 있다
- 가능한 선택을 하나씩 시도한다
- 선택 후 다시 되돌아와야 한다
- 순열/조합/부분집합/N-Queen 문제다
