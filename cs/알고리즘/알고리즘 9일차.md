# 2. 가중치 그래프

가중치 그래프는 간선에 비용이 있는 그래프입니다.

예:

```text
1 --(5)-- 2
1 --(2)-- 3
3 --(1)-- 2
```

여기서 괄호 안 숫자가 비용입니다.

일반 BFS는 모든 간선 비용이 같을 때 최단거리를 구하는 데 적합합니다.

간선 비용이 다르면 다익스트라 같은 알고리즘을 사용해야 합니다.

---

```java
class Node {
    int to;
    int cost;

    Node(int to, int cost) {
        this.to = to;
        this.cost = cost;
```

```java
List<List<Node>> graph = new ArrayList<>();

```

# 4. 다익스트라란?

다익스트라는 한 시작점에서 다른 모든 정점까지의 최단거리를 구하는 알고리즘입니다.

조건:

```text
간선 비용이 음수가 아니어야 한다.
```

대표 사용:

- 특정 도시에서 모든 도시까지 최단거리
- 네트워크 지연 시간
- 최소 비용 경로
- 가중치가 있는 길 찾기

---

# 5. 다익스트라 핵심 아이디어

현재까지 가장 거리가 짧은 정점을 먼저 확정합니다.

이를 위해 PriorityQueue를 사용합니다.

```text
1. 시작점 거리 = 0
2. 나머지 거리 = 무한대
3. 가장 가까운 정점을 꺼냄
4. 그 정점에서 갈 수 있는 곳의 거리를 갱신
5. 반복
```

---

# 6. 다익스트라 코드

```java
import java.util.*;

public class Main {
    static class Node {
        int to;
        int cost;

        Node(int to, int cost) {
            this.to = to;
            this.cost = cost;
        }
    }

    static int n = 5;
    static List<List<Node>> graph = new ArrayList<>();
    static int[] dist;

    public static void dijkstra(int start) {
        PriorityQueue<Node> pq = new PriorityQueue<>(
            (a, b) -> Integer.compare(a.cost, b.cost)
        );

        Arrays.fill(dist, Integer.MAX_VALUE);

        dist[start] = 0;
        pq.offer(new Node(start, 0));

        while (!pq.isEmpty()) {
            Node cur = pq.poll();

            int now = cur.to;
            int cost = cur.cost;

            if (cost > dist[now]) {
                continue;
            }

            for (Node next : graph.get(now)) {
                int newCost = dist[now] + next.cost;

                if (newCost < dist[next.to]) {
                    dist[next.to] = newCost;
                    pq.offer(new Node(next.to, newCost));
                }
            }
        }
    }

    public static void main(String[] args) {
        dist = new int[n + 1];

        for (int i = 0; i <= n; i++) {
            graph.add(new ArrayList<>());
        }

        graph.get(1).add(new Node(2, 2));
        graph.get(1).add(new Node(3, 5));
        graph.get(2).add(new Node(3, 1));
        graph.get(2).add(new Node(4, 2));
        graph.get(3).add(new Node(4, 3));
        graph.get(4).add(new Node(5, 1));

        dijkstra(1);

        System.out.println(Arrays.toString(dist));
    }
}
```

# 7. 다익스트라 코드 설명

```java
PriorityQueue<Node> pq = new PriorityQueue<>(
    (a, b) -> Integer.compare(a.cost, b.cost)
);
```

거리가 가장 짧은 노드를 먼저 꺼내기 위해 우선순위큐를 사용합니다.

```java
Arrays.fill(dist, Integer.MAX_VALUE);
```

처음에는 모든 거리를 무한대로 둡니다.

```java
dist[start] = 0;
```

시작점에서 시작점까지의 거리는 0입니다.

```java
pq.offer(new Node(start, 0));
```

시작 노드를 큐에 넣습니다.

```java
Node cur = pq.poll();
```

현재 가장 가까운 노드를 꺼냅니다.

```java
if (cost > dist[now]) {
    continue;
}
```

이미 더 짧은 경로로 처리된 노드라면 무시합니다.

이 조건이 없으면 불필요한 연산이 많아집니다.

```java
int newCost = dist[now] + next.cost;
```

현재 노드를 거쳐 next로 가는 비용입니다.

```java
if (newCost < dist[next.to])
```

더 짧은 경로를 찾았다면 갱신합니다.

```java
dist[next.to] = newCost;
pq.offer(new Node(next.to, newCost));
```

새 거리로 갱신하고 우선순위큐에 넣습니다.

이 경우 dist를 long으로 둡니다.


```java

```

특히 비용 합이 21억을 넘을 가능성이 있으면 long을 고려해야 합니다.

# 9. 플로이드 워셜

플로이드 워셜은 모든 정점에서 모든 정점까지의 최단거리를 구하는 알고리즘입니다.

특징:

```text
모든 쌍 최단거리
시간복잡도 O(N^3)
```

정점 수가 작을 때 사용합니다.

예를 들어 N이 100 정도면 가능하지만, N이 10000이면 불가능합니다.

---

# 10. 플로이드 워셜 점화식

```text
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

의미:

```text
i에서 j로 바로 가는 것보다
i에서 k를 거쳐 j로 가는 것이 더 짧으면 갱신
```

---

# 11. 플로이드 워셜 코드

```java
import java.util.*;

public class Main {
    static final int INF = 1_000_000_000;

    public static void main(String[] args) {
        int n = 4;

        int[][] dist = new int[n + 1][n + 1];

        for (int i = 1; i <= n; i++) {
            Arrays.fill(dist[i], INF);
            dist[i][i] = 0;
        }

        dist[1][2] = 4;
        dist[1][3] = 2;
        dist[3][2] = 1;
        dist[2][4] = 3;
        dist[3][4] = 7;

        for (int k = 1; k <= n; k++) {
            for (int i = 1; i <= n; i++) {
                for (int j = 1; j <= n; j++) {
                    if (dist[i][k] == INF || dist[k][j] == INF) {
                        continue;
                    }

                    dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);
                }
            }
        }

        System.out.println(dist[1][4]);
    }
}
```

---

# 12. 플로이드 워셜 코드 설명

```java
Arrays.fill(dist[i], INF);
```

처음에는 갈 수 없다고 표시합니다.

```java
dist[i][i] = 0;
```

자기 자신으로 가는 비용은 0입니다.

```java
for (int k = 1; k <= n; k++)
```

거쳐가는 정점입니다.

```java
for (int i = 1; i <= n; i++)
```

출발 정점입니다.

```java
for (int j = 1; j <= n; j++)
```

도착 정점입니다.

반복문 순서는 보통 k, i, j 순서로 씁니다.

```java
dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);
```

k를 거쳐 가는 경로가 더 짧으면 갱신합니다.

# 13. 다익스트라 vs 플로이드 워셜

| 구분       | 다익스트라              | 플로이드 워셜               |
| ---------- | ----------------------- | --------------------------- |
| 목적       | 한 시작점에서 모든 정점 | 모든 정점 쌍                |
| 시간복잡도 | O(E log V)              | O(V^3)                      |
| 자료구조   | 인접리스트 + PQ         | 2차원 배열                  |
| 음수 간선  | 불가능                  | 가능하지만 음수 사이클 주의 |
| 정점 수    | 큰 그래프 가능          | 작은 그래프                 |

# 14. 벨만 포드 개념

벨만 포드는 한 시작점에서 모든 정점까지의 최단거리를 구합니다.

다익스트라와 다르게 음수 간선이 있어도 사용할 수 있습니다.

또한 음수 사이클을 감지할 수 있습니다.

단점은 느립니다.

```text
시간복잡도 O(VE)
```

개념적으로는 모든 간선을 V-1번 반복해서 완화합니다.

# 15. 위상정렬

위상정렬은 방향 그래프에서 선후관계를 만족하도록 정점을 나열하는 알고리즘입니다.

조건:

```text
방향 그래프
사이클이 없어야 함
```

이런 그래프를 DAG라고 합니다.

```text
DAG = Directed Acyclic Graph
```

대표 문제:

- 선수과목
- 작업 순서
- 빌드 순서
- 프로젝트 작업 관계

# 16. 진입차수

진입차수는 어떤 정점으로 들어오는 간선의 개수입니다.

예:

```text
1 -> 3
2 -> 3
```

3번 정점의 진입차수는 2입니다.

위상정렬은 진입차수가 0인 정점부터 처리합니다.

---

# 17. 위상정렬 코드

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        int n = 4;

        List<List<Integer>> graph = new ArrayList<>();
        int[] indegree = new int[n + 1];

        for (int i = 0; i <= n; i++) {
            graph.add(new ArrayList<>());
        }

        graph.get(1).add(2);
        indegree[2]++;

        graph.get(1).add(3);
        indegree[3]++;

        graph.get(2).add(4);
        indegree[4]++;

        graph.get(3).add(4);
        indegree[4]++;

        Queue<Integer> q = new LinkedList<>();

        for (int i = 1; i <= n; i++) {
            if (indegree[i] == 0) {
                q.offer(i);
            }
        }

        List<Integer> result = new ArrayList<>();

        while (!q.isEmpty()) {
            int now = q.poll();
            result.add(now);

            for (int next : graph.get(now)) {
                indegree[next]--;

                if (indegree[next] == 0) {
                    q.offer(next);
                }
            }
        }

        System.out.println(result);
    }
}
```

---

# 18. 위상정렬 코드 설명

```java
int[] indegree = new int[n + 1];
```

각 정점의 진입차수를 저장합니다.

```java
graph.get(1).add(2);
indegree[2]++;
```

1에서 2로 가는 간선이 있으므로 2의 진입차수를 증가시킵니다.

```java
if (indegree[i] == 0)
```

먼저 처리할 수 있는 정점입니다.

```java
int now = q.poll();
```

현재 처리할 정점을 꺼냅니다.

```java
indegree[next]--;
```

현재 정점을 처리했으므로 next로 들어가는 간선 하나가 사라진 것처럼 봅니다.

```java
if (indegree[next] == 0)
```

next의 모든 선행 조건이 끝났다면 큐에 넣습니다.

# 19. 위상정렬 사이클 판별

위상정렬 결과의 개수가 n보다 작으면 사이클이 있는 것입니다.

```java
if (result.size() != n) {
    System.out.println("사이클 존재");
}
```

사이클이 있으면 진입차수가 0이 되는 정점이 더 이상 나오지 않습니다.

# 20. MST란?

MST는 Minimum Spanning Tree입니다.

최소 신장 트리라고 합니다.

의미:

```text
모든 정점을 연결하면서
간선 비용 합이 최소인 트리
```

조건:

- 모든 정점 연결
- 사이클 없음
- 간선 수는 N-1개

대표 알고리즘:

- 크루스칼
- 프림

여기서는 크루스칼을 다룹니다.

---

# 21. 크루스칼 알고리즘

크루스칼은 간선을 비용이 작은 순서대로 정렬한 뒤, 사이클이 생기지 않으면 선택합니다.

핵심 자료구조:

```text
Union Find
```

절차:

```text
1. 간선을 비용 기준 오름차순 정렬
2. 작은 간선부터 확인
3. 두 정점이 다른 집합이면 선택
4. union으로 합침
5. N-1개 선택하면 종료
```

---

# 22. 크루스칼 코드

```java
import java.util.*;

public class Main {
    static class Edge {
        int from;
        int to;
        int cost;

        Edge(int from, int to, int cost) {
            this.from = from;
            this.to = to;
            this.cost = cost;
        }
    }

    static int[] parent;

    public static int find(int x) {
        if (parent[x] == x) {
            return x;
        }

        return parent[x] = find(parent[x]);
    }

    public static void union(int a, int b) {
        int rootA = find(a);
        int rootB = find(b);

        if (rootA != rootB) {
            parent[rootB] = rootA;
        }
    }

    public static void main(String[] args) {
        int n = 4;

        parent = new int[n + 1];

        for (int i = 1; i <= n; i++) {
            parent[i] = i;
        }

        List<Edge> edges = new ArrayList<>();

        edges.add(new Edge(1, 2, 1));
        edges.add(new Edge(1, 3, 3));
        edges.add(new Edge(2, 3, 2));
        edges.add(new Edge(2, 4, 4));
        edges.add(new Edge(3, 4, 5));

        edges.sort((a, b) -> Integer.compare(a.cost, b.cost));

        int total = 0;
        int count = 0;

        for (Edge edge : edges) {
            if (find(edge.from) != find(edge.to)) {
                union(edge.from, edge.to);
                total += edge.cost;
                count++;
            }

            if (count == n - 1) {
                break;
            }
        }

        System.out.println(total);
    }
}
```

---

# 23. 크루스칼 코드 설명

```java
edges.sort((a, b) -> Integer.compare(a.cost, b.cost));
```

간선을 비용 기준 오름차순으로 정렬합니다.

```java
if (find(edge.from) != find(edge.to))
```

두 정점이 아직 같은 집합이 아니라면 사이클이 생기지 않습니다.

```java
union(edge.from, edge.to);
```

두 정점을 같은 집합으로 합칩니다.

```java
total += edge.cost;
```

선택한 간선의 비용을 누적합니다.

```java
count++;
```

선택한 간선 수를 증가시킵니다.

```java
if (count == n - 1)
```

MST는 간선이 N-1개면 완성됩니다.

# 24. 알고리즘 선택 기준

## BFS

```text
간선 비용이 모두 같고 최단거리
```

## 다익스트라

```text
간선 비용이 다르고 음수가 없음
한 시작점 기준 최단거리
```

## 플로이드 워셜

```text
모든 정점 쌍 최단거리
정점 수가 작음
```

## 벨만 포드

```text
음수 간선 존재
음수 사이클 확인 필요
```

## 위상정렬

```text
선후관계
방향 그래프
사이클 없음
```

## 크루스칼

```text
모든 정점 연결 최소 비용
MST
```

# 25. 자주 틀리는 부분

## 1. 다익스트라에 음수 간선 사용

다익스트라는 음수 간선이 있으면 사용할 수 없습니다.

음수 간선이 있으면 벨만 포드를 고려합니다.

## 2. 다익스트라에서 오래된 거리 무시 안 함

```java
if (cost > dist[now]) continue;
```

이 코드가 없으면 불필요한 처리가 많아집니다.

## 3. INF가 너무 작음

```java
int INF = 100000;
```

간선 비용 합이 더 커질 수 있으면 틀립니다.

넉넉하게 둡니다.

```java
int INF = 1_000_000_000;
```

또는 long을 사용합니다.

## 4. 플로이드 워셜 반복문 순서

보통 다음 순서를 사용합니다.

```java
for (k)
  for (i)
    for (j)
```

k가 가장 바깥입니다.

## 5. 위상정렬에서 진입차수 감소 누락

```java
indegree[next]--;
```

이걸 빼면 다음 정점이 큐에 들어가지 않습니다.

## 6. 크루스칼에서 Union Find 미사용

간선을 정렬만 하고 사이클 체크를 안 하면 MST가 아닐 수 있습니다.

## 7. 무방향 그래프 양방향 추가 누락

다익스트라에서도 무방향이면 양쪽에 넣어야 합니다.

```java
graph.get(a).add(new Node(b, cost));
graph.get(b).add(new Node(a, cost));
```

# 26. 시간복잡도 정리

| 알고리즘      | 시간복잡도 |
| ------------- | ---------: |
| BFS 최단거리  |   O(V + E) |
| 다익스트라    | O(E log V) |
| 플로이드 워셜 |     O(V^3) |
| 벨만 포드     |      O(VE) |
| 위상정렬      |   O(V + E) |
| 크루스칼      | O(E log E) |
