# 2. 정렬이란?

정렬은 데이터를 특정 기준에 따라 순서대로 배치하는 것입니다.

예를 들어 다음 배열이 있다고 합시다.

```text

```

오름차순 정렬 결과는 다음과 같습니다.

```text

```

내림차순 정렬 결과는 다음과 같습니다.

```text
[5, 4, 3, 2, 1]
```

# 4. 정렬 시간복잡도

Java의 `Arrays.sort()`는 타입에 따라 내부 방식이 다를 수 있지만, 알고리즘 문제에서는 일반적으로 다음처럼 생각하면 됩니다.

| 작업 | 시간복잡도 |
| ---- | ---------: |
| 정렬 | O(N log N) |

정렬이 들어가는 문제는 보통 O(N log N) 정도의 시간이 든다고 생각하면 됩니다.

```java
        Arrays.sort(arr, Collections.reverseOrder());

        System.out.println(Arrays.toString(arr));
    }
}
```

## 출력

```text
[5, 4, 3, 2, 1]
```

## 코드 설명

```java
Integer[] arr = {5, 1, 4, 2, 3};
```

`int[]`가 아니라 `Integer[]`입니다.

```java
Arrays.sort(arr, Collections.reverseOrder());
```

기본 오름차순 기준을 반대로 바꿔 내림차순으로 정렬합니다.

# 12. 2차원 배열 두 번째 값 기준 정렬

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        int[][] arr = {
            {1, 80},
            {2, 60},
            {3, 90}
        };

        Arrays.sort(arr, (a, b) -> Integer.compare(a[1], b[1]));

        for (int[] x : arr) {
            System.out.println(Arrays.toString(x));
        }
    }
}
```

## 출력

```text
[2, 60]
[1, 80]
[3, 90]
```

## 코드 설명

```java
Arrays.sort(arr, (a, b) -> Integer.compare(a[1], b[1]));
```

`a[1]`, `b[1]`을 비교합니다.

즉, 각 행의 두 번째 값을 기준으로 오름차순 정렬합니다.

# 13. 2차원 배열 첫 번째 값 오름차순, 같으면 두 번째 값 내림차순

```java
int[][] arr = {
    {1, 80},
    {2, 60},
    {1, 90},
    {2, 70}
};

Arrays.sort(arr, (a, b) -> {
    if (a[0] == b[0]) {
        return Integer.compare(b[1], a[1]);
    }

    return Integer.compare(a[0], b[0]);
});
```

## 정렬 기준

1. 첫 번째 값 기준 오름차순
2. 첫 번째 값이 같으면 두 번째 값 기준 내림차순

## 코드 설명

```java
if (a[0] == b[0])
```

첫 번째 값이 같다면:

```java
return Integer.compare(b[1], a[1]);
```

두 번째 값을 내림차순으로 정렬합니다.

```java
return Integer.compare(a[0], b[0]);
```

첫 번째 값이 다르면 첫 번째 값 기준 오름차순입니다.

# 16. 이분탐색(Binary Search)

이분탐색은 정렬된 데이터에서 원하는 값을 빠르게 찾는 알고리즘입니다.

중요한 조건:

```text
반드시 정렬되어 있어야 한다.
```

정렬되지 않은 배열에 이분탐색을 쓰면 결과가 잘못됩니다.

---

# 17. 이분탐색 개념

예를 들어 정렬된 배열이 있습니다.

```text
[1, 3, 5, 7, 9]
```

target이 7이라고 합시다.

1. 가운데 값 5 확인
2. 7은 5보다 크므로 오른쪽 탐색
3. 오른쪽 구간에서 가운데 값 7 확인
4. 찾음

이렇게 매번 탐색 범위를 절반으로 줄입니다.

시간복잡도는 O(log N)입니다.

---

# 18. 이분탐색 기본 코드

```java
public static boolean binarySearch(int[] arr, int target) {
    int left = 0;
    int right = arr.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (arr[mid] == target) {
            return true;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return false;
}
```

## 코드 설명

```java
int left = 0;
int right = arr.length - 1;
```

탐색 범위를 배열 전체로 잡습니다.

```java
while (left <= right)
```

탐색 범위가 남아 있는 동안 반복합니다.

```java
int mid = left + (right - left) / 2;
```

중간 위치를 계산합니다.

```java
if (arr[mid] == target)
```

중간 값이 찾는 값이면 성공입니다.

```java
else if (arr[mid] < target)
```

중간 값이 target보다 작으면 왼쪽 구간은 볼 필요가 없습니다.

따라서:

```java
left = mid + 1;
```

오른쪽 구간으로 이동합니다.

```java
else
```

중간 값이 target보다 크면 오른쪽 구간은 볼 필요가 없습니다.

따라서:

```java
right = mid - 1;
```

왼쪽 구간으로 이동합니다.

---

# 19. mid 계산에서 오버플로우 방지

다음 방식은 값이 매우 클 때 오버플로우 위험이 있습니다.

```java
int mid = (left + right) / 2;
```

안전한 방식:

```java
int mid = left + (right - left) / 2;
```

알고리즘 문제에서는 습관적으로 안전한 방식을 쓰는 것이 좋습니다.

# 21. lower bound

lower bound는 target 이상이 처음 나오는 위치입니다.

예:

```text
arr = [1, 2, 2, 2, 3, 4]
target = 2
```

2 이상이 처음 나오는 위치는 index 1입니다.

---

# 22. lower bound 코드

```java
public static int lowerBound(int[] arr, int target) {
    int left = 0;
    int right = arr.length;

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (arr[mid] >= target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }

    return left;
}
```

## 코드 설명

```java
int right = arr.length;
```

기본 이분탐색과 다르게 `arr.length - 1`이 아니라 `arr.length`를 사용합니다.

탐색 범위를 `[left, right)` 형태로 보기 때문입니다.

```java
while (left < right)
```

left와 right가 만날 때까지 반복합니다.

```java
if (arr[mid] >= target)
```

mid 위치의 값이 target 이상이면 정답 후보입니다.

하지만 더 왼쪽에 처음 등장하는 위치가 있을 수 있으므로:

```java
right = mid;
```

왼쪽으로 범위를 줄입니다.

```java
else
```

mid 값이 target보다 작으면 정답이 될 수 없습니다.

```java
left = mid + 1;
```

오른쪽으로 이동합니다.

# 23. upper bound

upper bound는 target보다 큰 값이 처음 나오는 위치입니다.

예:

```text
arr = [1, 2, 2, 2, 3, 4]
target = 2
```

2보다 큰 값이 처음 나오는 위치는 index 4입니다.

---

# 24. upper bound 코드

```java
public static int upperBound(int[] arr, int target) {
    int left = 0;
    int right = arr.length;

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (arr[mid] > target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }

    return left;
}
```

## lower bound와 차이

lower bound:

```java
arr[mid] >= target
```

upper bound:

```java
arr[mid] > target
```

이 차이가 핵심입니다.

# 26. 이분탐색을 정답 찾기에 쓰는 경우

이분탐색은 단순히 배열에서 값을 찾는 데만 쓰지 않습니다.

다음과 같은 문제에도 사용됩니다.

```text
조건을 만족하는 최소값 찾기
조건을 만족하는 최대값 찾기
```

예:

- 랜선 자르기
- 나무 자르기
- 공유기 설치
- 예산 배정
- 최소 시간 찾기

이런 문제는 보통 다음 구조입니다.

```java
while (left <= right) {
    int mid = left + (right - left) / 2;

    if (condition(mid)) {
        answer = mid;
        right = mid - 1;
    } else {
        left = mid + 1;
    }
}
```

# 27. 투포인터(Two Pointer)

투포인터는 두 개의 포인터를 움직이면서 조건을 만족하는 값을 찾는 방법입니다.

보통 다음 상황에서 사용합니다.

- 정렬된 배열에서 두 수의 합 찾기
- 연속 부분합
- 구간 길이 조절
- 중복 제거
- 양쪽 끝에서 비교

---

# 28. 두 수의 합: 정렬 + 투포인터

배열에서 두 수의 합이 target이 되는지 확인합니다.

```java
public static boolean twoSum(int[] arr, int target) {
    Arrays.sort(arr);

    int left = 0;
    int right = arr.length - 1;

    while (left < right) {
        int sum = arr[left] + arr[right];

        if (sum == target) {
            return true;
        } else if (sum < target) {
            left++;
        } else {
            right--;
        }
    }

    return false;
}
```

## 코드 설명

```java
Arrays.sort(arr);
```

투포인터를 사용하기 위해 배열을 정렬합니다.

```java
int left = 0;
int right = arr.length - 1;
```

왼쪽 끝과 오른쪽 끝에서 시작합니다.

```java
int sum = arr[left] + arr[right];
```

두 포인터가 가리키는 값의 합을 구합니다.

```java
if (sum == target)
```

목표값이면 성공입니다.

```java
else if (sum < target)
```

합이 너무 작으면 더 큰 값이 필요합니다.

정렬되어 있으므로 left를 오른쪽으로 이동합니다.

```java
left++;
```

```java
else
```

합이 너무 크면 더 작은 값이 필요합니다.

right를 왼쪽으로 이동합니다.

```java
right--;
```

# 30. 연속 부분합 투포인터

양수 배열에서 연속 부분합이 target 이상이 되는 최소 길이를 구하는 문제입니다.

```java
public static int minLength(int[] arr, int target) {
    int left = 0;
    int sum = 0;
    int answer = Integer.MAX_VALUE;

    for (int right = 0; right < arr.length; right++) {
        sum += arr[right];

        while (sum >= target) {
            answer = Math.min(answer, right - left + 1);
            sum -= arr[left];
            left++;
        }
    }

    return answer == Integer.MAX_VALUE ? 0 : answer;
}
```

## 코드 설명

```java
int left = 0;
int sum = 0;
```

현재 구간의 시작 위치와 합입니다.

```java
for (int right = 0; right < arr.length; right++)
```

right를 오른쪽으로 한 칸씩 확장합니다.

```java
sum += arr[right];
```

현재 값을 구간합에 더합니다.

```java
while (sum >= target)
```

조건을 만족하는 동안 왼쪽을 줄여봅니다.

```java
answer = Math.min(answer, right - left + 1);
```

현재 구간 길이로 정답을 갱신합니다.

```java
sum -= arr[left];
left++;
```

왼쪽 값을 제거하고 left를 오른쪽으로 이동합니다.

## 주의

이 방식은 보통 모든 값이 양수일 때 안정적으로 사용합니다.

음수가 섞이면 합이 단조롭게 변하지 않기 때문에 다른 방식이 필요할 수 있습니다.

# 31. 슬라이딩 윈도우

슬라이딩 윈도우는 고정된 길이의 구간을 이동하면서 값을 계산하는 방법입니다.

예:

```text
배열에서 길이가 3인 연속 구간의 최대합 구하기
```

---

# 32. 슬라이딩 윈도우 기본 코드

```java
public static int maxWindowSum(int[] arr, int k) {
    int sum = 0;

    for (int i = 0; i < k; i++) {
        sum += arr[i];
    }

    int max = sum;

    for (int i = k; i < arr.length; i++) {
        sum += arr[i];
        sum -= arr[i - k];

        max = Math.max(max, sum);
    }

    return max;
}
```

## 코드 설명

```java
for (int i = 0; i < k; i++)
```

처음 k개 구간의 합을 구합니다.

```java
int max = sum;
```

첫 번째 구간합을 최댓값으로 초기화합니다.

```java
for (int i = k; i < arr.length; i++)
```

윈도우를 한 칸씩 오른쪽으로 이동합니다.

```java
sum += arr[i];
```

새로 들어온 오른쪽 값을 더합니다.

```java
sum -= arr[i - k];
```

윈도우에서 빠지는 왼쪽 값을 뺍니다.

```java
max = Math.max(max, sum);
```

최댓값을 갱신합니다.

# 33. 투포인터와 슬라이딩 윈도우 차이

| 구분        | 투포인터               | 슬라이딩 윈도우 |
| ----------- | ---------------------- | --------------- |
| 구간 길이   | 변할 수 있음           | 보통 고정       |
| 포인터 이동 | 조건에 따라 이동       | 일정하게 이동   |
| 대표 문제   | 합 조건 만족 최소 길이 | 길이 k 최대합   |
| 핵심        | 조건 조절              | 구간 유지       |

# 34. 정렬 + 이분탐색 + 투포인터 선택 기준

## 정렬을 의심할 때

- 순서가 중요하다
- 가장 작은/큰 값을 비교한다
- 중복을 모아야 한다
- 좌표/시간/점수 기준 정리가 필요하다

## 이분탐색을 의심할 때

- 정렬된 배열에서 찾는다
- 탐색 범위가 크다
- 조건을 만족하는 최소/최대값을 찾는다
- `가능/불가능` 판정이 있다

## 투포인터를 의심할 때

- 양쪽 끝에서 비교한다
- 두 수의 합을 찾는다
- 연속 구간을 조절한다
- 한 번의 순회로 해결 가능해 보인다

## 슬라이딩 윈도우를 의심할 때

- 고정 길이 구간
- 연속된 k개
- 구간합을 계속 갱신
- 한 칸씩 이동

# 35. 자주 틀리는 부분

## 1. 정렬 안 하고 이분탐색 사용

```java
int[] arr = {5, 1, 3};

binarySearch(arr, 3);
```

이분탐색 전에는 반드시 정렬해야 합니다.

```java
Arrays.sort(arr);
binarySearch(arr, 3);
```

## 2. left, right 범위 실수

일반 이분탐색:

```java
int right = arr.length - 1;
while (left <= right)
```

lower bound:

```java
int right = arr.length;
while (left < right)
```

두 방식을 섞으면 오류가 나기 쉽습니다.

## 3. mid 계산 오버플로우

```java
int mid = (left + right) / 2;
```

더 안전한 방식:

```java
int mid = left + (right - left) / 2;
```

## 4. Comparator에서 빼기 사용

```java
(a, b) -> a - b
```

오버플로우 위험이 있습니다.

안전한 방식:

```java
(a, b) -> Integer.compare(a, b)
```

## 5. 투포인터에서 정렬 조건 누락

두 수의 합 문제에서 양끝 투포인터를 쓰려면 보통 정렬이 필요합니다.

## 6. 슬라이딩 윈도우에서 빠지는 값 인덱스 실수

```java
sum -= arr[i - k];
```

현재 i에서 새 값이 들어왔으므로 빠지는 값은 `i - k`입니다.

# 36. 시간복잡도 정리

| 알고리즘        | 시간복잡도 |
| --------------- | ---------: |
| 정렬            | O(N log N) |
| 기본 이분탐색   |   O(log N) |
| lower bound     |   O(log N) |
| upper bound     |   O(log N) |
| 투포인터        |       O(N) |
| 슬라이딩 윈도우 |       O(N) |
| 정렬 + 투포인터 | O(N log N) |
