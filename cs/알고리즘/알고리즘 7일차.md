# 2. DP란?

DP는 Dynamic Programming의 줄임말입니다.

큰 문제를 작은 문제로 나누고,
작은 문제의 답을 저장해서
같은 계산을 반복하지 않는 방식입니다.

핵심:

```text
중복되는 작은 문제
작은 문제 답으로 큰 문제 해결
```

---

# 3. 왜 DP를 쓰나?

예를 들어 피보나치 재귀:

```java
fib(n) = fib(n-1) + fib(n-2)
```

fib(5)를 구할 때:

```text
fib(5)
 -> fib(4), fib(3)
fib(4)
 -> fib(3), fib(2)
```

fib(3)이 여러 번 계산됩니다.

DP는 이미 계산한 값을 저장해서 재사용합니다.

---

# 4. 메모이제이션

이미 계산한 값을 저장하는 기법입니다.

```java
static int[] dp = new int[100];
```

값이 있으면 재사용합니다.

---

# 5. 탑다운 방식

재귀 + 메모이제이션 방식입니다.

```java
public static int fib(int n) {
    if (n <= 1) return n;
    if (dp[n] != 0) return dp[n];

    dp[n] = fib(n - 1) + fib(n - 2);
    return dp[n];
}
```

---

# 6. 바텀업 방식

작은 값부터 차례대로 채워가는 방식입니다.

```java
dp[0] = 0;
dp[1] = 1;

for (int i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
}
```

# 7. 피보나치 전체 코드

```java
public class Main {
    public static void main(String[] args) {
        int n = 10;
        int[] dp = new int[n + 1];

        dp[0] = 0;
        dp[1] = 1;

        for (int i = 2; i <= n; i++) {
            dp[i] = dp[i - 1] + dp[i - 2];
        }

        System.out.println(dp[n]);
    }
}
```

# 8. 점화식이란?

현재 상태를 이전 상태로 표현하는 식입니다.

예:

```text
dp[i] = dp[i-1] + dp[i-2]
```

현재 값이 이전 두 값으로 결정됩니다.

DP 핵심은 점화식을 찾는 것입니다.

---

# 9. 계단 오르기

한 번에 1칸 또는 2칸 이동 가능.

N칸까지 가는 방법 수.

```text
dp[i] = dp[i-1] + dp[i-2]
```

## 코드

```java
int n = 5;
int[] dp = new int[n + 1];

dp[1] = 1;
dp[2] = 2;

for (int i = 3; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
}
```

---

# 10. 최소 비용 계단

각 칸 비용이 있을 때 최소 비용.

```text
dp[i] = min(dp[i-1], dp[i-2]) + cost[i]
```

---

# 11. 1로 만들기

N에서 시작해서:

- 3으로 나누기
- 2로 나누기
- 1 빼기

최소 횟수로 1 만들기.

## 점화식

```text
dp[i] = dp[i-1] + 1
if i % 2 == 0 -> min(dp[i], dp[i/2] + 1)
if i % 3 == 0 -> min(dp[i], dp[i/3] + 1)
```

## 코드

```java
int n = 10;
int[] dp = new int[n + 1];

for (int i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + 1;

    if (i % 2 == 0) {
        dp[i] = Math.min(dp[i], dp[i / 2] + 1);
    }

    if (i % 3 == 0) {
        dp[i] = Math.min(dp[i], dp[i / 3] + 1);
    }
}
```

# 12. 동전 거슬러주기 (최소 개수)

동전 종류:

```text
1, 4, 6
```

금액 8 만들기 최소 개수.

## 점화식

```text
dp[i] = min(dp[i], dp[i-coin] + 1)
```

## 코드

```java
int[] coin = {1, 4, 6};
int target = 8;

int[] dp = new int[target + 1];
Arrays.fill(dp, 1000000);

dp[0] = 0;

for (int c : coin) {
    for (int i = c; i <= target; i++) {
        dp[i] = Math.min(dp[i], dp[i - c] + 1);
    }
}
```

---

# 13. 동전 경우의 수

동전으로 만드는 경우의 수.

```java
dp[0] = 1;

for (int c : coin) {
    for (int i = c; i <= target; i++) {
        dp[i] += dp[i - c];
    }
}
```

# 14. LIS 기초

가장 긴 증가 부분 수열.

```text
1 5 2 3 4
=> 1 2 3 4
길이 4
```

## O(N^2) 코드

```java
int[] arr = {1,5,2,3,4};
int n = arr.length;
int[] dp = new int[n];

Arrays.fill(dp, 1);

for (int i = 0; i < n; i++) {
    for (int j = 0; j < i; j++) {
        if (arr[j] < arr[i]) {
            dp[i] = Math.max(dp[i], dp[j] + 1);
        }
    }
}
```

# 15. 배낭 문제 기초

무게 제한 K 안에서 최대 가치.

```text
물건:
무게 3 가치 5
무게 4 가치 6
```

## 2차원 DP

```java
int[][] dp = new int[n + 1][K + 1];
```

점화식:

```text
현재 물건 안 넣음
현재 물건 넣음
```

---

# 16. 0/1 Knapsack 코드

```java
for (int i = 1; i <= n; i++) {
    for (int w = 1; w <= K; w++) {
        dp[i][w] = dp[i-1][w];

        if (w >= weight[i]) {
            dp[i][w] = Math.max(
                dp[i][w],
                dp[i-1][w-weight[i]] + value[i]
            );
        }
    }
}
```

# 17. DP 배열 의미 정하기

DP는 먼저 의미를 정해야 합니다.

예:

```text
dp[i] = i번째까지 봤을 때 최대값
dp[i] = i를 만드는 최소 횟수
dp[i][j] = i번째까지 사용해서 j 만들기
```

의미가 정해져야 점화식이 나옵니다.

# 19. 초기값(Base Case)

매우 중요합니다.

예:

```java
dp[0] = 0;
dp[1] = 1;
```

초기값 틀리면 전체가 틀립니다.

---

# 20. 1차원 DP vs 2차원 DP

| 형태        | 의미    |
| ----------- | ------- |
| dp[i]       | 한 상태 |
| dp[i][j]    | 두 상태 |
| dp[i][j][k] | 세 상태 |

예:

```text
행/열
날짜/상태
개수/합계
```

# 21. 공간 최적화

피보나치는 이전 2개만 필요합니다.

```java
int a = 0;
int b = 1;

for (int i = 2; i <= n; i++) {
    int c = a + b;
    a = b;
    b = c;
}
```

배열 없이 가능.

# 22. 자주 틀리는 부분

## 1. 초기값 누락

```java
dp[1] = 1;
```

이런 기본값 빠뜨리면 오류.

## 2. 배열 크기 부족

```java
new int[n + 1]
```

1-index면 보통 n+1 필요.

## 3. 반복 순서 실수

동전 문제는 순서에 따라 의미 달라짐.

## 4. 최소값 초기화 안 함

```java
Arrays.fill(dp, INF);
```

필요할 때 많음.

## 5. 점화식 잘못 세움

현재 상태가 무엇인지 먼저 정의해야 함.
