# 2. 트리란?

트리는 사이클이 없는 계층형 자료구조입니다.

예:

```text
        1
      /   \
     2     3
    / \
   4   5
```

용어:

| 용어     | 의미                              |
| -------- | --------------------------------- |
| 루트     | 가장 위 노드                      |
| 부모     | 위쪽에 연결된 노드                |
| 자식     | 아래쪽에 연결된 노드              |
| 리프     | 자식이 없는 노드                  |
| 깊이     | 루트에서 떨어진 거리              |
| 서브트리 | 특정 노드를 루트로 하는 작은 트리 |

---

# 3. 트리와 그래프 차이

트리는 그래프의 한 종류입니다.

| 구분      | 그래프       | 트리                   |
| --------- | ------------ | ---------------------- |
| 사이클    | 있을 수 있음 | 없음                   |
| 연결      | 끊길 수 있음 | 보통 모두 연결         |
| 부모/자식 | 없음         | 있음                   |
| 경로      | 여러 개 가능 | 두 노드 사이 경로 하나 |

트리는 보통 다음 조건을 만족합니다.

```text
정점 N개
간선 N-1개
사이클 없음
모든 정점 연결
```

---

# 4. 트리 표현 방법

트리도 그래프처럼 인접리스트로 표현할 수 있습니다.

```java
List<List<Integer>> tree = new ArrayList<>();

for (int i = 0; i <= n; i++) {
    tree.add(new ArrayList<>());
}

tree.get(1).add(2);
tree.get(2).add(1);

tree.get(1).add(3);
tree.get(3).add(1);
```

무방향 트리라면 양방향으로 저장합니다.

# 5. 부모 배열 만들기

트리에서 각 노드의 부모를 저장할 수 있습니다.

```java
static List<List<Integer>> tree = new ArrayList<>();
static int[] parent;
static boolean[] visited;

public static void dfs(int now) {
    visited[now] = true;

    for (int next : tree.get(now)) {
        if (!visited[next]) {
            parent[next] = now;
            dfs(next);
        }
    }
}
```

## 코드 설명

```java
parent[next] = now;
```

next 노드의 부모를 now로 저장합니다.

예:

```text
1 - 2
1 - 3
```

1에서 DFS를 시작하면:

```text
parent[2] = 1
parent[3] = 1
```

# 8. 전위 순회

전위 순회는 다음 순서입니다.

```text
루트 -> 왼쪽 -> 오른쪽
```

## 코드

```java
public static void preorder(Node node) {
    if (node == null) {
        return;
    }

    System.out.print(node.value + " ");
    preorder(node.left);
    preorder(node.right);
}
```

## 코드 설명

```java
if (node == null) return;
```

노드가 없으면 종료합니다.

```java
System.out.print(node.value);
```

현재 노드를 먼저 방문합니다.

```java
preorder(node.left);
preorder(node.right);
```

왼쪽, 오른쪽 순서로 방문합니다.

# 9. 중위 순회

중위 순회는 다음 순서입니다.

```text
왼쪽 -> 루트 -> 오른쪽
```

## 코드

```java
public static void inorder(Node node) {
    if (node == null) {
        return;
    }

    inorder(node.left);
    System.out.print(node.value + " ");
    inorder(node.right);
}
```

이진 탐색 트리에서 중위 순회를 하면 값이 정렬된 순서로 나옵니다.

# 10. 후위 순회

후위 순회는 다음 순서입니다.

```text
왼쪽 -> 오른쪽 -> 루트
```

## 코드

```java
public static void postorder(Node node) {
    if (node == null) {
        return;
    }

    postorder(node.left);
    postorder(node.right);
    System.out.print(node.value + " ");
}
```

후위 순회는 하위 노드를 먼저 처리하고 부모를 처리해야 할 때 사용합니다.

# 11. 레벨 순회

레벨 순회는 위에서 아래로, 왼쪽에서 오른쪽으로 방문합니다.

BFS를 사용합니다.

```java
public static void levelOrder(Node root) {
    Queue<Node> q = new LinkedList<>();

    q.offer(root);

    while (!q.isEmpty()) {
        Node now = q.poll();

        System.out.print(now.value + " ");

        if (now.left != null) {
            q.offer(now.left);
        }

        if (now.right != null) {
            q.offer(now.right);
        }
    }
}
```

## 코드 설명

```java
Queue<Node> q = new LinkedList<>();
```

방문할 노드를 저장합니다.

```java
q.offer(root);
```

루트부터 시작합니다.

```java
Node now = q.poll();
```

현재 레벨의 노드를 꺼냅니다.

```java
q.offer(now.left);
q.offer(now.right);
```

자식 노드를 큐에 넣습니다.

# 12. 힙이란?

힙은 우선순위가 높은 값을 빠르게 꺼내기 위한 자료구조입니다.

Java에서는 보통 `PriorityQueue`를 사용합니다.

기본은 최소힙입니다.

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();

pq.offer(3);
pq.offer(1);
pq.offer(2);

System.out.println(pq.poll()); // 1
```

---

# 16. Union Find란?

Union Find는 서로소 집합 자료구조입니다.

다음 기능을 빠르게 처리합니다.

- 두 원소가 같은 집합인지 확인
- 두 집합을 합치기

대표 문제:

- 사이클 판별
- 네트워크 연결
- 친구 관계
- 크루스칼 알고리즘
- 집합 개수 세기

---

# 17. Union Find 기본 배열

```java
static int[] parent;

public static void makeSet(int n) {
    parent = new int[n + 1];

    for (int i = 1; i <= n; i++) {
        parent[i] = i;
    }
}
```

## 코드 설명

```java
parent[i] = i;
```

처음에는 자기 자신이 대표입니다.

예:

```text
parent[1] = 1
parent[2] = 2
parent[3] = 3
```

각자 따로 집합입니다.

# 18. find

find는 어떤 원소의 대표를 찾습니다.

```java
public static int find(int x) {
    if (parent[x] == x) {
        return x;
    }

    return parent[x] = find(parent[x]);
}
```

## 코드 설명

```java
if (parent[x] == x)
```

자기 자신이 부모라면 대표입니다.

```java
return parent[x] = find(parent[x]);
```

부모를 따라가서 대표를 찾고, 그 결과를 바로 parent[x]에 저장합니다.

이것을 경로 압축이라고 합니다.

---

# 19. union

union은 두 집합을 합칩니다.

```java
public static void union(int a, int b) {
    int rootA = find(a);
    int rootB = find(b);

    if (rootA != rootB) {
        parent[rootB] = rootA;
    }
}
```

## 코드 설명

```java
int rootA = find(a);
int rootB = find(b);
```

두 원소의 대표를 찾습니다.

```java
if (rootA != rootB)
```

대표가 다르면 서로 다른 집합입니다.

```java
parent[rootB] = rootA;
```

rootB 집합을 rootA 밑으로 붙입니다.

---

# 20. 같은 집합인지 확인

```java
if (find(a) == find(b)) {
    System.out.println("같은 집합");
} else {
    System.out.println("다른 집합");
}
```

---

# 21. Union Find 전체 코드

```java
public class Main {
    static int[] parent;

    public static void makeSet(int n) {
        parent = new int[n + 1];

        for (int i = 1; i <= n; i++) {
            parent[i] = i;
        }
    }

    public static int find(int x) {
        if (parent[x] == x) {
            return x;
        }

        return parent[x] = find(parent[x]);
    }

    public static void union(int a, int b) {
        int rootA = find(a);
        int rootB = find(b);

        if (rootA != rootB) {
            parent[rootB] = rootA;
        }
    }

    public static void main(String[] args) {
        makeSet(5);

        union(1, 2);
        union(3, 4);

        System.out.println(find(1) == find(2));
        System.out.println(find(1) == find(3));
    }
}
```

---

# 22. Union Find로 사이클 판별

무방향 그래프에서 간선을 하나씩 보면서 이미 같은 집합이면 사이클입니다.

```java
boolean hasCycle = false;

for (int[] edge : edges) {
    int a = edge[0];
    int b = edge[1];

    if (find(a) == find(b)) {
        hasCycle = true;
        break;
    }

    union(a, b);
}
```

## 코드 설명

```java
if (find(a) == find(b))
```

이미 같은 집합인데 또 연결하려 한다면 사이클이 생깁니다.

# 23. Segment Tree란?

Segment Tree는 구간 정보를 빠르게 구하고 수정하기 위한 자료구조입니다.

대표 기능:

- 구간합
- 구간 최소값
- 구간 최대값

배열에서 구간합을 자주 구하고 값도 자주 바뀐다면 세그먼트 트리를 사용합니다.

---

# 24. Segment Tree가 필요한 이유

누적합은 구간합 조회가 빠릅니다.

```text
구간합 조회 O(1)
```

하지만 중간 값이 바뀌면 누적합 배열을 다시 만들어야 할 수 있습니다.

Segment Tree는:

```text
구간합 조회 O(log N)
값 수정 O(log N)
```

둘 다 빠르게 처리할 수 있습니다.

---

# 25. Segment Tree 구간합 기본 구조

```java
static long[] arr;
static long[] tree;

public static long init(int node, int start, int end) {
    if (start == end) {
        return tree[node] = arr[start];
    }

    int mid = (start + end) / 2;

    return tree[node] =
        init(node * 2, start, mid)
        + init(node * 2 + 1, mid + 1, end);
}
```

## 코드 설명

```java
node
```

현재 세그먼트 트리의 노드 번호입니다.

```java
start, end
```

현재 노드가 담당하는 배열 구간입니다.

```java
if (start == end)
```

리프 노드입니다. 원본 배열 값을 저장합니다.

```java
node * 2
node * 2 + 1
```

왼쪽 자식, 오른쪽 자식입니다.

---

# 26. Segment Tree 구간합 조회

```java
public static long query(int node, int start, int end, int left, int right) {
    if (right < start || end < left) {
        return 0;
    }

    if (left <= start && end <= right) {
        return tree[node];
    }

    int mid = (start + end) / 2;

    return query(node * 2, start, mid, left, right)
         + query(node * 2 + 1, mid + 1, end, left, right);
}
```

## 코드 설명

```java
if (right < start || end < left)
```

현재 구간과 찾는 구간이 전혀 겹치지 않습니다.

구간합에서는 0을 반환합니다.

```java
if (left <= start && end <= right)
```

현재 구간이 찾는 구간에 완전히 포함됩니다.

이 경우 tree[node]를 그대로 사용합니다.

```java
return query(...) + query(...);
```

일부만 겹치면 왼쪽 자식과 오른쪽 자식을 나누어 탐색합니다.

---

# 27. Segment Tree 값 수정

```java
public static void update(int node, int start, int end, int index, long diff) {
    if (index < start || index > end) {
        return;
    }

    tree[node] += diff;

    if (start == end) {
        return;
    }

    int mid = (start + end) / 2;

    update(node * 2, start, mid, index, diff);
    update(node * 2 + 1, mid + 1, end, index, diff);
}
```

## 코드 설명

```java
if (index < start || index > end)
```

수정할 index가 현재 구간 밖이면 무시합니다.

```java
tree[node] += diff;
```

현재 구간합에 변경 차이를 반영합니다.

```java
update(...)
```

자식 노드에도 내려가며 값을 수정합니다.

# 28. Fenwick Tree란?

Fenwick Tree는 Binary Indexed Tree라고도 합니다.

구간합과 값 수정을 빠르게 처리합니다.

Segment Tree보다 코드가 짧고 메모리를 적게 씁니다.

대표 기능:

```text
prefix sum
point update
range sum
```

---

# 29. Fenwick Tree 핵심

Fenwick Tree는 인덱스의 마지막 1비트를 이용합니다.

```java
i += i & -i
i -= i & -i
```

처음 보면 어렵지만, 알고리즘 문제에서는 패턴처럼 외워도 됩니다.

---

# 30. Fenwick Tree update

```java
static long[] tree;
static int n;

public static void update(int index, long diff) {
    while (index <= n) {
        tree[index] += diff;
        index += index & -index;
    }
}
```

## 코드 설명

```java
tree[index] += diff;
```

현재 위치에 변경값을 반영합니다.

```java
index += index & -index;
```

다음 담당 구간으로 이동합니다.

---

# 31. Fenwick Tree prefix sum

```java
public static long sum(int index) {
    long result = 0;

    while (index > 0) {
        result += tree[index];
        index -= index & -index;
    }

    return result;
}
```

## 코드 설명

```java
result += tree[index];
```

현재 인덱스가 담당하는 구간합을 더합니다.

```java
index -= index & -index;
```

이전 담당 구간으로 이동합니다.

---

# 32. Fenwick Tree 구간합

```java
public static long rangeSum(int left, int right) {
    return sum(right) - sum(left - 1);
}
```

## 코드 설명

```java
sum(right)
```

1번부터 right까지 합입니다.

```java
sum(left - 1)
```

1번부터 left-1까지 합입니다.

둘을 빼면 left부터 right까지의 합이 됩니다.

---

# 33. Segment Tree vs Fenwick Tree

| 구분           | Segment Tree | Fenwick Tree      |
| -------------- | ------------ | ----------------- |
| 코드 길이      | 김           | 짧음              |
| 구간합         | 가능         | 가능              |
| 값 수정        | 가능         | 가능              |
| 구간 최소/최대 | 가능         | 일반적으로 어려움 |
| 메모리         | 약 4N        | 약 N              |
| 난이도         | 높음         | 중간              |

구간합만 필요하면 Fenwick Tree가 편할 수 있습니다.

구간 최소/최대까지 필요하면 Segment Tree를 고려합니다.

# 34. 자주 틀리는 부분

## 1. 트리에서 부모 방향으로 다시 감

무방향 트리를 인접리스트로 만들면 부모로 다시 갈 수 있습니다.

따라서 visited가 필요합니다.

```java
if (!visited[next]) {
    dfs(next);
}
```

또는 parent를 이용해서 막을 수 있습니다.

## 2. 순회에서 null 체크 누락

```java
if (node == null) {
    return;
}
```

이 조건이 없으면 NullPointerException이 발생할 수 있습니다.

## 3. PriorityQueue 내부가 완전 정렬이라고 착각

PriorityQueue는 poll할 때 우선순위가 가장 높은 값이 나옵니다.

내부 전체가 정렬된 배열은 아닙니다.

## 4. Union Find에서 find 안 하고 union

잘못된 코드:

```java
parent[b] = a;
```

정상 코드:

```java
parent[find(b)] = find(a);
```

대표끼리 연결해야 합니다.

## 5. Segment Tree 배열 크기 부족

보통 다음처럼 넉넉하게 잡습니다.

```java
tree = new long[n * 4];
```

## 6. Fenwick Tree는 1-index 기반

Fenwick Tree는 보통 인덱스를 1부터 사용합니다.

0부터 쓰면 무한 루프나 잘못된 결과가 나올 수 있습니다.

# 35. 시간복잡도 정리

| 자료구조      | 주요 연산    | 시간복잡도 |
| ------------- | ------------ | ---------: |
| 트리 DFS      | 전체 탐색    |       O(N) |
| 트리 BFS      | 전체 탐색    |       O(N) |
| PriorityQueue | 삽입/삭제    |   O(log N) |
| Union Find    | find/union   |  거의 O(1) |
| Segment Tree  | query/update |   O(log N) |
| Fenwick Tree  | sum/update   |   O(log N) |

Union Find는 경로 압축을 사용하면 거의 상수 시간에 가깝게 동작합니다.

# 36. 실전 선택 기준

## 트리를 의심할 때

- 부모/자식 관계
- 계층 구조
- 루트가 있음
- 사이클이 없음
- 노드 N개, 간선 N-1개

## PriorityQueue를 의심할 때

- 계속 최솟값/최댓값을 꺼내야 함
- 우선순위가 있음
- 최소 비용 선택
- 다익스트라
- 카드 정렬

## Union Find를 의심할 때

- 두 원소가 같은 그룹인지 확인
- 그룹 합치기
- 네트워크 연결
- 사이클 판별
- 크루스칼

## Segment Tree를 의심할 때

- 구간합/최솟값/최댓값 질의가 많음
- 중간 값 수정도 많음
- 누적합으로는 부족함

## Fenwick Tree를 의심할 때

- 구간합만 필요
- 값 수정도 필요
- 세그먼트 트리보다 짧은 코드 원함
