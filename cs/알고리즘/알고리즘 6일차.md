# 2. 그래프란?

그래프는 정점과 간선으로 이루어진 자료구조입니다.

```text
정점(Vertex): 데이터가 있는 지점
간선(Edge): 정점과 정점을 연결하는 선
```

예:

```text
1 - 2
1 - 3
2 - 4
3 - 5
```

여기서 정점은 1, 2, 3, 4, 5입니다.

간선은 다음과 같습니다.

```text
1과 2 연결
1과 3 연결
2와 4 연결
3과 5 연결
```

---

# 3. 그래프 표현 방법

그래프는 대표적으로 두 가지 방식으로 표현합니다.

1. 인접행렬
2. 인접리스트

---

# 4. 인접행렬

인접행렬은 2차원 배열로 연결 여부를 저장합니다.

```java
int[][] graph = new int[6][6];

graph[1][2] = 1;
graph[2][1] = 1;

graph[1][3] = 1;
graph[3][1] = 1;
```

## 코드 설명

```java
graph[1][2] = 1;
```

1번 정점과 2번 정점이 연결되어 있다는 뜻입니다.

무방향 그래프라면 반대 방향도 저장해야 합니다.

```java
graph[2][1] = 1;
```

## 장점

- 두 정점이 연결되어 있는지 O(1)에 확인 가능

## 단점

- 정점 수가 많으면 메모리 낭비가 큼
- 간선이 적은 그래프에서는 비효율적

---

# 5. 인접리스트

인접리스트는 각 정점마다 연결된 정점 목록을 저장합니다.

```java
List<List<Integer>> graph = new ArrayList<>();

for (int i = 0; i <= 5; i++) {
    graph.add(new ArrayList<>());
}

graph.get(1).add(2);
graph.get(2).add(1);

graph.get(1).add(3);
graph.get(3).add(1);
```

## 코드 설명

```java
List<List<Integer>> graph = new ArrayList<>();
```

그래프 전체를 저장할 리스트입니다.

```java
graph.add(new ArrayList<>());
```

각 정점마다 연결 목록을 저장할 리스트를 하나씩 만듭니다.

```java
graph.get(1).add(2);
```

1번 정점에서 2번 정점으로 갈 수 있다는 뜻입니다.

무방향 그래프라면 반대 방향도 추가합니다.

```java
graph.get(2).add(1);
```

## 장점

- 간선이 적을 때 메모리 효율적
- 실제 알고리즘 문제에서 가장 많이 사용

## 단점

- 두 정점이 연결되었는지 바로 확인하려면 리스트를 탐색해야 함

---

# 6. 인접행렬 vs 인접리스트

| 구분         | 인접행렬   | 인접리스트         |
| ------------ | ---------- | ------------------ |
| 구조         | 2차원 배열 | 리스트 안의 리스트 |
| 연결 확인    | O(1)       | O(차수)            |
| 메모리       | O(V^2)     | O(V + E)           |
| 간선 적을 때 | 비효율적   | 효율적             |
| 실전 사용    | 적은 편    | 매우 많음          |

V는 정점 수, E는 간선 수입니다.

# 7. DFS란?

DFS는 Depth First Search입니다.

깊이 우선 탐색이라고 합니다.

한 방향으로 갈 수 있을 만큼 깊게 들어간 뒤, 더 이상 갈 곳이 없으면 되돌아옵니다.

DFS는 보통 다음 방식으로 구현합니다.

- 재귀
- 스택

---

# 8. DFS 재귀 기본 코드

```java
import java.util.*;

public class Main {
    static List<List<Integer>> graph = new ArrayList<>();
    static boolean[] visited;

    public static void dfs(int now) {
        visited[now] = true;
        System.out.println(now);

        for (int next : graph.get(now)) {
            if (!visited[next]) {
                dfs(next);
            }
        }
    }

    public static void main(String[] args) {
        int n = 5;

        visited = new boolean[n + 1];

        for (int i = 0; i <= n; i++) {
            graph.add(new ArrayList<>());
        }

        graph.get(1).add(2);
        graph.get(2).add(1);

        graph.get(1).add(3);
        graph.get(3).add(1);

        graph.get(2).add(4);
        graph.get(4).add(2);

        graph.get(3).add(5);
        graph.get(5).add(3);

        dfs(1);
    }
}
```

## 코드 설명

```java
static List<List<Integer>> graph = new ArrayList<>();
```

그래프를 저장합니다.

```java
static boolean[] visited;
```

각 정점을 방문했는지 저장합니다.

```java
visited[now] = true;
```

현재 정점을 방문 처리합니다.

```java
for (int next : graph.get(now))
```

현재 정점과 연결된 다음 정점들을 하나씩 확인합니다.

```java
if (!visited[next])
```

아직 방문하지 않은 정점이면 이동합니다.

```java
dfs(next);
```

다음 정점으로 깊게 들어갑니다.

---

# 9. DFS 방문 순서

그래프:

```text
1 - 2 - 4
|
3 - 5
```

1번에서 시작하면 연결 리스트 순서에 따라 방문 순서가 달라질 수 있습니다.

예:

```text
1 -> 2 -> 4 -> 3 -> 5
```

DFS는 한 방향으로 깊게 들어가는 것이 핵심입니다.

---

# 10. DFS 스택 구현

재귀 대신 Stack 또는 Deque를 사용할 수도 있습니다.

```java
public static void dfsStack(int start) {
    Deque<Integer> stack = new ArrayDeque<>();
    stack.push(start);

    while (!stack.isEmpty()) {
        int now = stack.pop();

        if (visited[now]) {
            continue;
        }

        visited[now] = true;
        System.out.println(now);

        for (int next : graph.get(now)) {
            if (!visited[next]) {
                stack.push(next);
            }
        }
    }
}
```

## 코드 설명

```java
Deque<Integer> stack = new ArrayDeque<>();
```

스택 역할을 할 Deque입니다.

```java
stack.push(start);
```

시작 정점을 넣습니다.

```java
int now = stack.pop();
```

가장 최근에 넣은 정점을 꺼냅니다.

```java
if (visited[now]) continue;
```

이미 방문했다면 건너뜁니다.

```java
stack.push(next);
```

방문할 후보를 스택에 넣습니다.

# 11. BFS란?

BFS는 Breadth First Search입니다.

너비 우선 탐색이라고 합니다.

가까운 정점부터 차례대로 방문합니다.

BFS는 보통 Queue로 구현합니다.

---

# 12. BFS 기본 코드

```java
import java.util.*;

public class Main {
    static List<List<Integer>> graph = new ArrayList<>();
    static boolean[] visited;

    public static void bfs(int start) {
        Queue<Integer> q = new LinkedList<>();

        q.offer(start);
        visited[start] = true;

        while (!q.isEmpty()) {
            int now = q.poll();

            System.out.println(now);

            for (int next : graph.get(now)) {
                if (!visited[next]) {
                    visited[next] = true;
                    q.offer(next);
                }
            }
        }
    }
}
```

## 코드 설명

```java
Queue<Integer> q = new LinkedList<>();
```

방문할 정점을 저장할 큐입니다.

```java
q.offer(start);
visited[start] = true;
```

시작 정점을 큐에 넣고 방문 처리합니다.

```java
while (!q.isEmpty())
```

큐가 빌 때까지 반복합니다.

```java
int now = q.poll();
```

큐에서 가장 먼저 들어온 정점을 꺼냅니다.

```java
for (int next : graph.get(now))
```

현재 정점과 연결된 정점을 확인합니다.

```java
visited[next] = true;
q.offer(next);
```

다음 정점을 방문 처리하고 큐에 넣습니다.

BFS에서는 보통 큐에 넣는 순간 방문 처리합니다.

---

# 13. BFS 방문 순서

그래프:

```text
1 - 2 - 4
|
3 - 5
```

1번에서 시작하면 보통 다음처럼 가까운 정점부터 방문합니다.

```text
1 -> 2 -> 3 -> 4 -> 5
```

BFS는 최단거리 문제에서 자주 사용됩니다.

단, 모든 간선의 비용이 같을 때 기본 BFS로 최단거리를 구할 수 있습니다.

---

# 14. DFS와 BFS 차이

| 구분        | DFS                 | BFS                 |
| ----------- | ------------------- | ------------------- |
| 의미        | 깊이 우선 탐색      | 너비 우선 탐색      |
| 자료구조    | 재귀/스택           | 큐                  |
| 방문 방식   | 깊게 먼저           | 가까운 곳 먼저      |
| 대표 문제   | 경로 탐색, 백트래킹 | 최단거리, 레벨 탐색 |
| 구현 난이도 | 재귀면 간단         | 큐 필요             |

# 15. 연결 요소 개수 세기

그래프가 여러 덩어리로 나뉘어 있을 때, 덩어리 개수를 세는 문제입니다.

```java
int count = 0;

for (int i = 1; i <= n; i++) {
    if (!visited[i]) {
        dfs(i);
        count++;
    }
}
```

## 코드 설명

```java
for (int i = 1; i <= n; i++)
```

모든 정점을 확인합니다.

```java
if (!visited[i])
```

아직 방문하지 않은 정점이면 새로운 연결 요소입니다.

```java
dfs(i);
```

그 정점과 연결된 모든 정점을 방문 처리합니다.

```java
count++;
```

연결 요소 개수를 1 증가시킵니다.

# 16. 2차원 격자 탐색

격자 문제도 그래프로 볼 수 있습니다.

예:

```text
1 1 0
0 1 0
1 1 1
```

1은 이동 가능, 0은 이동 불가라고 생각할 수 있습니다.

상하좌우 이동을 위해 방향 배열을 사용합니다.

```java
int[] dx = {-1, 1, 0, 0};
int[] dy = {0, 0, -1, 1};
```

---

# 17. 격자 DFS

```java
static int n = 3;
static int m = 3;

static int[][] board = {
    {1, 1, 0},
    {0, 1, 0},
    {1, 1, 1}
};

static boolean[][] visited = new boolean[n][m];

static int[] dx = {-1, 1, 0, 0};
static int[] dy = {0, 0, -1, 1};

public static void dfs(int x, int y) {
    visited[x][y] = true;

    for (int dir = 0; dir < 4; dir++) {
        int nx = x + dx[dir];
        int ny = y + dy[dir];

        if (nx < 0 || ny < 0 || nx >= n || ny >= m) {
            continue;
        }

        if (visited[nx][ny]) {
            continue;
        }

        if (board[nx][ny] == 0) {
            continue;
        }

        dfs(nx, ny);
    }
}
```

## 코드 설명

```java
visited[x][y] = true;
```

현재 칸을 방문 처리합니다.

```java
for (int dir = 0; dir < 4; dir++)
```

상하좌우 4방향을 확인합니다.

```java
int nx = x + dx[dir];
int ny = y + dy[dir];
```

다음 위치를 계산합니다.

```java
if (nx < 0 || ny < 0 || nx >= n || ny >= m)
```

격자 밖이면 무시합니다.

```java
if (visited[nx][ny])
```

이미 방문했다면 무시합니다.

```java
if (board[nx][ny] == 0)
```

이동할 수 없는 칸이면 무시합니다.

```java
dfs(nx, ny);
```

이동 가능한 칸이면 재귀로 탐색합니다.

# 18. 격자 BFS 최단거리

격자에서 최단거리를 구할 때는 BFS를 많이 사용합니다.

```java
import java.util.*;

public class Main {
    static int n = 3;
    static int m = 3;

    static int[][] board = {
        {1, 1, 0},
        {0, 1, 1},
        {0, 0, 1}
    };

    static int[][] dist = new int[n][m];

    static int[] dx = {-1, 1, 0, 0};
    static int[] dy = {0, 0, -1, 1};

    public static void bfs(int sx, int sy) {
        Queue<int[]> q = new LinkedList<>();

        q.offer(new int[]{sx, sy});
        dist[sx][sy] = 1;

        while (!q.isEmpty()) {
            int[] cur = q.poll();

            int x = cur[0];
            int y = cur[1];

            for (int dir = 0; dir < 4; dir++) {
                int nx = x + dx[dir];
                int ny = y + dy[dir];

                if (nx < 0 || ny < 0 || nx >= n || ny >= m) {
                    continue;
                }

                if (board[nx][ny] == 0) {
                    continue;
                }

                if (dist[nx][ny] != 0) {
                    continue;
                }

                dist[nx][ny] = dist[x][y] + 1;
                q.offer(new int[]{nx, ny});
            }
        }
    }
}
```

## 코드 설명

```java
Queue<int[]> q = new LinkedList<>();
```

좌표를 저장하는 큐입니다.

```java
q.offer(new int[]{sx, sy});
```

시작 좌표를 큐에 넣습니다.

```java
dist[sx][sy] = 1;
```

시작 지점의 거리를 1로 둡니다.

문제에 따라 0으로 둘 수도 있습니다.

```java
int[] cur = q.poll();
int x = cur[0];
int y = cur[1];
```

현재 좌표를 꺼냅니다.

```java
dist[nx][ny] = dist[x][y] + 1;
```

다음 칸의 거리는 현재 칸 거리보다 1 큽니다.

```java
q.offer(new int[]{nx, ny});
```

다음 칸을 큐에 넣습니다.

# 19. BFS에서 dist 배열을 visited처럼 쓰기

BFS에서는 `visited` 배열 대신 `dist` 배열을 방문 여부로 사용할 수 있습니다.

```java
if (dist[nx][ny] != 0) {
    continue;
}
```

이미 거리가 기록된 칸이면 방문한 칸입니다.

단, 시작 거리를 0으로 두는 문제에서는 별도의 visited 배열을 쓰는 것이 더 안전할 수 있습니다.

# 22. DFS/BFS 자주 틀리는 부분

## 1. 방문 처리 누락

```java
dfs(next);
```

방문 처리를 하지 않으면 무한 반복이 생길 수 있습니다.

정상:

```java
visited[next] = true;
dfs(next);
```

재귀 DFS에서는 함수 시작에서 방문 처리해도 됩니다.

## 2. BFS 방문 처리를 늦게 함

좋은 방식:

```java
visited[next] = true;
q.offer(next);
```

나쁜 방식:

```java
q.offer(next);
// 나중에 꺼낼 때 방문 처리
```

큐에 같은 노드가 여러 번 들어갈 수 있습니다.

## 3. 무방향 그래프에서 한쪽만 추가

```java
graph.get(a).add(b);
```

무방향이면 반대도 추가해야 합니다.

```java
graph.get(b).add(a);
```

## 4. 1번 시작인데 배열 크기를 n으로 만듦

정점 번호가 1부터 n이면 배열 크기는 n+1이 편합니다.

```java
boolean[] visited = new boolean[n + 1];
```

## 5. 격자 범위 체크 누락

```java
if (nx < 0 || ny < 0 || nx >= n || ny >= m) continue;
```

이 조건은 거의 필수입니다.

# 23. 시간복잡도 정리

| 알고리즘           |          시간복잡도 |
| ------------------ | ------------------: |
| 인접행렬 DFS/BFS   |              O(V^2) |
| 인접리스트 DFS/BFS |            O(V + E) |
| 격자 DFS/BFS       |           O(N \* M) |
| 인접리스트 정렬    | 간선 수에 따라 다름 |

V는 정점 수, E는 간선 수입니다.

격자는 칸의 개수가 N _ M이므로 각 칸을 한 번씩 방문하면 O(N _ M)입니다.

---

# 24. DFS/BFS 선택 기준

## DFS를 의심할 때

- 모든 경로를 끝까지 탐색
- 백트래킹
- 연결 요소 찾기
- 재귀적으로 깊게 들어가는 문제
- 섬의 개수
- 사이클 탐지 기초

## BFS를 의심할 때

- 최단거리
- 최소 이동 횟수
- 가까운 곳부터 탐색
- 레벨 단위 탐색
- 미로 탐색
- 토마토 익히기 유형
