# 2. Stack

Stack은 마지막에 넣은 값이 가장 먼저 나오는 자료구조입니다.

이런 구조를 LIFO라고 합니다.

```text
LIFO = Last In First Out
```

즉, 나중에 들어간 값이 먼저 나옵니다.

---

# 3. Stack 기본 코드

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Stack<Integer> stack = new Stack<>();

        stack.push(10);
        stack.push(20);
        stack.push(30);

        System.out.println(stack.pop());
        System.out.println(stack.pop());
        System.out.println(stack.peek());
    }
}
```

## 출력

```text
30
20
10
```

## 코드 설명

```java
Stack<Integer> stack = new Stack<>();
```

정수를 저장하는 스택을 생성합니다.

```java
stack.push(10);
stack.push(20);
stack.push(30);
```

스택에 값을 차례대로 넣습니다.

스택 상태:

```text
아래쪽 [10, 20, 30] 위쪽
```

```java
stack.pop()
```

맨 위의 값을 꺼내고 제거합니다.

첫 번째 `pop()`은 30입니다.

두 번째 `pop()`은 20입니다.

```java
stack.peek()
```

맨 위의 값을 확인만 하고 제거하지 않습니다.

현재 남은 값은 10이므로 `peek()` 결과는 10입니다.

# 6. 괄호 검사 문제

문자열이 올바른 괄호인지 확인하는 문제입니다.

예:

```text
()       올바름
(())     올바름
(()      올바르지 않음
())      올바르지 않음
```

---

# 7. 괄호 검사 코드

```java
import java.util.*;

public class Main {
    public static boolean isValid(String s) {
        Stack<Character> stack = new Stack<>();

        for (int i = 0; i < s.length(); i++) {
            char ch = s.charAt(i);

            if (ch == '(') {
                stack.push(ch);
            } else if (ch == ')') {
                if (stack.isEmpty()) {
                    return false;
                }

                stack.pop();
            }
        }

        return stack.isEmpty();
    }

    public static void main(String[] args) {
        System.out.println(isValid("()"));
        System.out.println(isValid("(())"));
        System.out.println(isValid("(()"));
        System.out.println(isValid("())"));
    }
}
```

## 코드 설명

```java
Stack<Character> stack = new Stack<>();
```

여는 괄호를 저장할 스택입니다.

```java
if (ch == '(') {
    stack.push(ch);
}
```

여는 괄호가 나오면 스택에 넣습니다.

```java
else if (ch == ')')
```

닫는 괄호가 나오면 이전에 열린 괄호가 있어야 합니다.

```java
if (stack.isEmpty()) {
    return false;
}
```

스택이 비어 있는데 닫는 괄호가 나왔다는 것은 매칭되는 여는 괄호가 없다는 뜻입니다.

```java
stack.pop();
```

매칭되는 여는 괄호 하나를 제거합니다.

```java
return stack.isEmpty();
```

모든 문자를 처리한 뒤 스택이 비어 있어야 올바른 괄호입니다.

스택에 여는 괄호가 남아 있으면 닫히지 않은 괄호가 있다는 뜻입니다.

# 8. 여러 종류 괄호 검사

괄호 종류가 여러 개일 수도 있습니다.

```text
()
{}
[]
```

예:

```text
({[]})   올바름
({[})    올바르지 않음
```

---

# 9. 여러 괄호 검사 코드

```java
import java.util.*;

public class Main {
    public static boolean isValid(String s) {
        Stack<Character> stack = new Stack<>();

        for (int i = 0; i < s.length(); i++) {
            char ch = s.charAt(i);

            if (ch == '(' || ch == '{' || ch == '[') {
                stack.push(ch);
            } else {
                if (stack.isEmpty()) {
                    return false;
                }

                char top = stack.pop();

                if (ch == ')' && top != '(') return false;
                if (ch == '}' && top != '{') return false;
                if (ch == ']' && top != '[') return false;
            }
        }

        return stack.isEmpty();
    }
}
```

## 코드 설명

```java
if (ch == '(' || ch == '{' || ch == '[')
```

여는 괄호면 스택에 넣습니다.

```java
char top = stack.pop();
```

닫는 괄호가 나오면 가장 최근 여는 괄호를 꺼냅니다.

```java
if (ch == ')' && top != '(') return false;
```

닫는 괄호와 여는 괄호의 종류가 맞는지 확인합니다.

# 11. Queue

Queue는 먼저 들어간 값이 먼저 나오는 자료구조입니다.

이런 구조를 FIFO라고 합니다.

```text
FIFO = First In First Out
```

즉, 먼저 들어온 값이 먼저 나갑니다.

---

```java

        q.offer(20);
        q.offer(30);

        System.out.println(q.poll());
        System.out.println(q.poll());
        System.out.println(q.peek());
    }
}
```

## 출력

```text

20
30
```

## 코드 설명

```java
Queue<Integer> q = new LinkedList<>();
```

```java

q.offer(20);
q.offer(30);
```

```text

```

q.poll()

큐의 앞쪽 값을 꺼내고 제거합니다.

# 16. BFS 기초

BFS는 가까운 곳부터 차례대로 탐색하는 알고리즘입니다.

보통 Queue를 사용합니다.

예를 들어 그래프가 다음과 같다고 합시다.

```text
1 - 2
1 - 3
2 - 4
3 - 5
```

1번에서 BFS를 시작하면 보통 다음 순서로 방문합니다.

```text
1 -> 2 -> 3 -> 4 -> 5
```

---

# 17. BFS 기본 코드

```java
import java.util.*;

public class Main {
    public static void bfs(int start, List<List<Integer>> graph, boolean[] visited) {
        Queue<Integer> q = new LinkedList<>();

        q.offer(start);
        visited[start] = true;

        while (!q.isEmpty()) {
            int now = q.poll();

            System.out.println(now);

            for (int next : graph.get(now)) {
                if (!visited[next]) {
                    visited[next] = true;
                    q.offer(next);
                }
            }
        }
    }
}
```

## 코드 설명

```java
Queue<Integer> q = new LinkedList<>();
```

방문할 노드를 저장할 큐입니다.

```java
q.offer(start);
visited[start] = true;
```

시작 노드를 큐에 넣고 방문 처리합니다.

```java
while (!q.isEmpty())
```

큐가 빌 때까지 반복합니다.

```java
int now = q.poll();
```

현재 방문할 노드를 꺼냅니다.

```java
for (int next : graph.get(now))
```

현재 노드와 연결된 다음 노드들을 확인합니다.

```java
if (!visited[next])
```

아직 방문하지 않은 노드라면:

```java
visited[next] = true;
q.offer(next);
```

방문 처리하고 큐에 넣습니다.

## 중요한 점

BFS에서는 보통 큐에 넣는 순간 방문 처리합니다.

그래야 같은 노드가 큐에 여러 번 들어가는 것을 막을 수 있습니다.

# 18. Deque

Deque는 Double Ended Queue의 줄임말입니다.

양쪽 끝에서 삽입과 삭제가 가능합니다.

```text
앞쪽 삽입 가능
앞쪽 삭제 가능
뒤쪽 삽입 가능
뒤쪽 삭제 가능
```

---

# 19. Deque 기본 코드

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Deque<Integer> deque = new ArrayDeque<>();

        deque.addFirst(10);
        deque.addLast(20);
        deque.addFirst(5);

        System.out.println(deque.pollFirst());
        System.out.println(deque.pollLast());
    }
}
```

## 출력

```text
5
20
```

## 코드 설명

```java
deque.addFirst(10);
```

앞쪽에 10을 넣습니다.

```java
deque.addLast(20);
```

뒤쪽에 20을 넣습니다.

```java
deque.addFirst(5);
```

앞쪽에 5를 넣습니다.

Deque 상태:

```text
앞 [5, 10, 20] 뒤
```

```java
deque.pollFirst()
```

앞쪽 값을 꺼냅니다. 결과는 5입니다.

```java
deque.pollLast()
```

뒤쪽 값을 꺼냅니다. 결과는 20입니다.

# 23. PriorityQueue

PriorityQueue는 우선순위가 높은 값부터 꺼내는 자료구조입니다.

Java의 `PriorityQueue`는 기본적으로 작은 값이 먼저 나옵니다.

즉, 최소힙입니다.

---

# 24. PriorityQueue 기본 코드

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        PriorityQueue<Integer> pq = new PriorityQueue<>();

        pq.offer(30);
        pq.offer(10);
        pq.offer(20);

        System.out.println(pq.poll());
        System.out.println(pq.poll());
        System.out.println(pq.poll());
    }
}
```

## 출력

```text
10
20
30
```

## 코드 설명

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
```

기본적으로 작은 값이 먼저 나오는 우선순위큐를 생성합니다.

```java
pq.offer(30);
pq.offer(10);
pq.offer(20);
```

값을 넣은 순서는 30, 10, 20입니다.

하지만 꺼낼 때는 작은 값부터 나옵니다.

```java
pq.poll()
```

현재 큐에서 가장 작은 값을 꺼냅니다.

---

# 25. PriorityQueue 시간복잡도

| 작업  | 시간복잡도 |
| ----- | ---------- |
| offer | O(log N)   |
| poll  | O(log N)   |
| peek  | O(1)       |

정렬을 매번 하는 것보다 효율적으로 최솟값이나 최댓값을 관리할 수 있습니다.

# 28. 객체 PriorityQueue

알고리즘에서는 비용, 거리, 우선순위 등을 함께 저장해야 하는 경우가 많습니다.

예를 들어 노드 번호와 비용을 함께 저장해봅니다.

```java
class Node {
    int index;
    int cost;

    Node(int index, int cost) {
        this.index = index;
        this.cost = cost;
    }
}
```

## 비용이 작은 순서대로 꺼내기

```java
PriorityQueue<Node> pq = new PriorityQueue<>(
    (a, b) -> Integer.compare(a.cost, b.cost)
);

pq.offer(new Node(1, 50));
pq.offer(new Node(2, 10));
pq.offer(new Node(3, 30));

System.out.println(pq.poll().index);
```

## 출력

```text
2
```

## 코드 설명

```java
(a, b) -> Integer.compare(a.cost, b.cost)
```

cost가 작은 Node가 먼저 나오도록 정렬 기준을 정합니다.

```java
pq.offer(new Node(2, 10));
```

index는 2, cost는 10인 객체를 넣습니다.

```java
pq.poll()
```

cost가 가장 작은 객체가 먼저 나옵니다.

# 30. 카드 정렬 예제

작은 묶음부터 합쳐야 전체 비용이 최소가 되는 문제입니다.

```java
import java.util.*;

public class Main {
    public static int minCost(int[] cards) {
        PriorityQueue<Integer> pq = new PriorityQueue<>();

        for (int card : cards) {
            pq.offer(card);
        }

        int total = 0;

        while (pq.size() > 1) {
            int a = pq.poll();
            int b = pq.poll();

            int sum = a + b;
            total += sum;

            pq.offer(sum);
        }

        return total;
    }

    public static void main(String[] args) {
        System.out.println(minCost(new int[]{10, 20, 40}));
    }
}
```

## 코드 설명

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
```

가장 작은 카드 묶음을 빠르게 꺼내기 위해 최소힙을 사용합니다.

```java
int a = pq.poll();
int b = pq.poll();
```

가장 작은 두 묶음을 꺼냅니다.

```java
int sum = a + b;
total += sum;
```

두 묶음을 합친 비용을 누적합니다.

```java
pq.offer(sum);
```

합쳐진 묶음을 다시 우선순위큐에 넣습니다.

## 왜 PriorityQueue가 필요한가

매번 가장 작은 두 값을 골라야 하기 때문입니다.

정렬 배열로 직접 관리하면 비효율적일 수 있습니다.

PriorityQueue를 쓰면 매번 O(log N)에 최소값을 꺼낼 수 있습니다.

---

# 31. Stack, Queue, Deque, PriorityQueue 비교

| 자료구조      | 꺼내는 기준      | 대표 사용                     |
| ------------- | ---------------- | ----------------------------- |
| Stack         | 마지막에 넣은 값 | 괄호, DFS, 최근 값            |
| Queue         | 먼저 넣은 값     | BFS, 대기열                   |
| Deque         | 양쪽 끝          | 슬라이딩 윈도우, 스택/큐 대체 |
| PriorityQueue | 우선순위         | 최소값/최대값, 다익스트라     |

---

# 32. 시간복잡도 정리

| 자료구조      |     삽입 |     삭제 | 조회 |
| ------------- | -------: | -------: | ---: |
| Stack         |     O(1) |     O(1) | O(1) |
| Queue         |     O(1) |     O(1) | O(1) |
| Deque         |     O(1) |     O(1) | O(1) |
| PriorityQueue | O(log N) | O(log N) | O(1) |

PriorityQueue에서 조회 O(1)은 `peek()` 기준입니다.

정렬된 전체 순서를 순회하는 것은 아닙니다.

# 33. 자주 틀리는 부분

## 1. 빈 Stack에서 pop

```java
stack.pop();
```

스택이 비어 있으면 예외가 발생합니다.

항상 확인하는 습관이 좋습니다.

```java
if (!stack.isEmpty()) {
    stack.pop();
}
```

## 2. 빈 Queue에서 poll

```java
q.poll();
```

큐가 비어 있으면 `null`이 나올 수 있습니다.

기본형 int로 바로 받으면 문제가 될 수 있습니다.

```java
Integer x = q.poll();
```

또는:

```java
if (!q.isEmpty()) {
    int x = q.poll();
}
```

## 3. PriorityQueue는 전체 정렬 상태가 아니다

```java
System.out.println(pq);
```

이렇게 출력했을 때 완전히 정렬된 것처럼 보이지 않을 수 있습니다.

PriorityQueue는 `poll()`할 때마다 우선순위가 가장 높은 값이 나오는 구조입니다.

전체 내부 배열이 정렬되어 있는 것은 아닙니다.

## 4. 최대힙 Comparator 오버플로우

```java
(a, b) -> b - a
```

값 차이가 매우 크면 오버플로우가 발생할 수 있습니다.

안전하게는:

```java
(a, b) -> Integer.compare(b, a)
```

## 5. BFS 방문 처리를 늦게 하는 실수

BFS에서는 보통 큐에 넣을 때 방문 처리합니다.

좋은 방식:

```java
visited[next] = true;
q.offer(next);
```

나쁜 방식:

```java
q.offer(next);
// 나중에 꺼낼 때 방문 처리
```

방문 처리를 늦게 하면 같은 노드가 큐에 여러 번 들어갈 수 있습니다.

# 34. 실전 선택 기준

## Stack을 써야 하는 경우

다음 키워드가 나오면 Stack을 의심합니다.

- 최근
- 되돌리기
- 괄호
- 짝 맞추기
- 왼쪽/오른쪽에서 가까운 큰 값
- 문자열 폭발

## Queue를 써야 하는 경우

다음 키워드가 나오면 Queue를 의심합니다.

- 순서대로 처리
- 먼저 들어온 것 먼저 처리
- BFS
- 최단 거리
- 레벨 탐색
- 대기열

## Deque를 써야 하는 경우

다음 키워드가 나오면 Deque를 의심합니다.

- 앞뒤로 넣고 빼기
- 양방향 처리
- 슬라이딩 윈도우 최댓값/최솟값
- 회전 큐

## PriorityQueue를 써야 하는 경우

다음 키워드가 나오면 PriorityQueue를 의심합니다.

- 가장 작은 값부터
- 가장 큰 값부터
- 최소 비용
- 우선순위
- 다익스트라
- 계속 최솟값을 꺼내야 함
