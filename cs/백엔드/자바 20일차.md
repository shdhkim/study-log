# Java Comparator.comparingInt 상세 설명

## 1. 문제의 코드

다음 코드가 있다고 가정한다.

``` java
String name = students.stream()
        .max(Comparator.comparingInt(Student::getScore))
        .get()
        .getName();
```

이 코드에서 핵심은 다음 부분이다.

``` java
Comparator.comparingInt(Student::getScore)
```

이 코드가 어떻게 동작하는지 단계별로 설명한다.

------------------------------------------------------------------------

# 2. 전체 흐름

전체 흐름은 다음과 같다.

    Student::getScore
    ↓
    람다식으로 변환
    ↓
    s -> s.getScore()
    ↓
    ToIntFunction<Student>
    ↓
    Comparator.comparingInt 호출
    ↓
    Comparator<Student> 생성
    ↓
    (s1, s2) -> Integer.compare(s1.getScore(), s2.getScore())

즉 최종적으로 **score 기준 Comparator가 생성된다.**

------------------------------------------------------------------------

# 3. Comparator.comparingInt 메소드

자바의 Comparator 클래스에는 다음과 같은 정적 메소드가 존재한다.

``` java
public static <T> Comparator<T> comparingInt(ToIntFunction<? super T> keyExtractor)
```

여기서 파라미터 타입은 다음과 같다.

    ToIntFunction<T>

즉

    T → int

형태의 함수를 전달해야 한다.

------------------------------------------------------------------------

# 4. ToIntFunction

ToIntFunction은 함수형 인터페이스이다.

``` java
@FunctionalInterface
public interface ToIntFunction<T> {

    int applyAsInt(T value);

}
```

즉 다음과 같은 의미이다.

    Student → int

Student 객체를 받아서 int 값을 반환하는 함수.

------------------------------------------------------------------------

# 5. 메서드 참조

다음 코드가 있다.

``` java
Student::getScore
```

이것은 **메서드 참조**이다.

컴파일러는 이것을 다음 람다식으로 변환한다.

``` java
s -> s.getScore()
```

즉

    Student -> int

함수이다.

------------------------------------------------------------------------

# 6. comparingInt 내부 동작

Comparator.comparingInt의 내부 구조를 단순화하면 다음과 같다.

``` java
public static <T> Comparator<T> comparingInt(ToIntFunction<? super T> keyExtractor) {

    return (c1, c2) ->
        Integer.compare(
            keyExtractor.applyAsInt(c1),
            keyExtractor.applyAsInt(c2)
        );
}
```

즉 다음 Comparator를 만들어서 반환한다.

``` java
Comparator<Student> c =
    (s1, s2) ->
        Integer.compare(
            s1.getScore(),
            s2.getScore()
        );
```

------------------------------------------------------------------------

# 7. 따라서 개념적으로 동일한 코드

다음 두 코드는 동일한 의미이다.

### 코드 1

``` java
Comparator.comparingInt(Student::getScore)
```

### 코드 2

``` java
(s1, s2) -> Integer.compare(s1.getScore(), s2.getScore())
```

즉 score 기준 Comparator가 생성된다.

------------------------------------------------------------------------

# 8. Stream max에서 사용

max 메소드는 다음 형태이다.

``` java
Optional<T> max(Comparator<? super T> comparator)
```

따라서 다음 코드가 가능하다.

``` java
students.stream()
        .max(Comparator.comparingInt(Student::getScore));
```

동작 과정

    Student 두 개 비교
    ↓
    score 비교
    ↓
    큰 값 선택

------------------------------------------------------------------------

# 9. Optional

max의 반환값은 Optional이다.

``` java
Optional<Student>
```

값이 없을 수도 있기 때문이다.

그래서 다음 코드가 사용된다.

``` java
.get()
```

Optional 내부 값을 꺼낸다.

------------------------------------------------------------------------

# 10. 전체 코드 동작 과정

    students
    ↓
    stream()
    ↓
    max(score 기준 comparator)
    ↓
    Optional<Student>
    ↓
    get()
    ↓
    Student
    ↓
    getName()
    ↓
    String

------------------------------------------------------------------------

# 11. 더 안전한 코드

실무에서는 다음 코드가 더 안전하다.

``` java
String name =
    students.stream()
        .max(Comparator.comparingInt(Student::getScore))
        .map(Student::getName)
        .orElse(null);
```

이 방식은 Optional 예외를 방지한다.

------------------------------------------------------------------------

# 12. 핵심 정리

  개념                 의미
  -------------------- --------------------------
  Student::getScore    메서드 참조
  s -\> s.getScore()   람다식
  ToIntFunction        T → int 함수
  comparingInt         Comparator 생성
  max                  최대값 찾기
  Optional             값이 없을 수도 있는 객체
