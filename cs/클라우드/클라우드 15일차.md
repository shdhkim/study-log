# 2. Load Balancer

## 역할

여러 서버로 트래픽을 분산하는 네트워크 컴포넌트

    Client
     ↓
    Load Balancer
     ↓
    Server1
    Server2
    Server3

## 주요 기능

  기능              설명
  ----------------- -----------------------
  트래픽 분산       여러 서버로 요청 분산
  Health Check      서버 상태 확인
  TLS termination   HTTPS 처리
  Failover          장애 서버 제외

## Nginx Load Balancer 예

``` nginx
upstream backend {
    server 10.0.1.5;
    server 10.0.1.6;
    server 10.0.1.7;
}

server {
    listen 80;

    location / {
        proxy_pass http://backend;
    }
}
```

# 5. Spring Cloud Gateway

Spring Boot 기반 API Gateway

## 의존성

Gradle

``` gradle
implementation 'org.springframework.cloud:spring-cloud-starter-gateway'
```

## Gateway 서버 구조

    Client
     ↓
    Spring Cloud Gateway
     ↓
    Microservices

## 라우팅 설정

``` yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/users/**
```

동작

    /users/** → user-service

# 7. Eureka Service Discovery

서비스 위치를 관리하는 서버

    Service Instance
          ↓
       Eureka Server

## 서비스 등록 설정

``` yaml
spring:
  application:
    name: user-service

eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka
```

서비스 시작 시

    POST /eureka/apps/USER-SERVICE

# 8. Eureka Heartbeat

서비스가 주기적으로 상태를 전송

    Service → Eureka

요청

    PUT /eureka/apps/USER-SERVICE/{instanceId}

기본 설정

  항목        기본값
  ----------- --------
  heartbeat   30초
  eviction    90초

# 9. Spring Boot Actuator

서비스 상태 확인 API

## Dependency

``` gradle
implementation 'org.springframework.boot:spring-boot-starter-actuator'
```

## health endpoint

    GET /actuator/health

응답

``` json
{
 "status": "UP"
}
```

## 상세 응답

``` json
{
 "status": "UP",
 "components": {
   "db": {
     "status": "UP"
   }
 }
}
```

------------------------------------------------------------------------

# 10. Eureka Health Check 설정

``` yaml
eureka:
  client:
    healthcheck:
      enabled: true
```

  설정    동작
  ------- --------------------
  false   heartbeat만 확인
  true    actuator 상태 반영

예

    DB DOWN → OUT_OF_SERVICE

# 14. Kubernetes vs Spring Cloud 차이

  기능                Spring Cloud
  ------------------- ----------------
  Service Discovery   Eureka
  Load Balancing      Client-side LB
  Health Check        Actuator
  Scaling             없음
  Networking          없음

------------------------------------------------------------------------

# 핵심 정리

    Kubernetes 없음
    → Spring Cloud가 MSA 인프라 담당

    Kubernetes 있음
    → Kubernetes가 인프라 담당
    → Spring Cloud 역할 축소
