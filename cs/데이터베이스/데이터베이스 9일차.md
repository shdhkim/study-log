# 트랜잭션 격리수준 & 동시성 이상현상 가이드 

> DB: Oracle 기준 예시 + 표준 SQL 병행

------------------------------------------------------------------------

## 공통 준비 테이블

``` sql
CREATE TABLE account (
  id NUMBER PRIMARY KEY,
  balance NUMBER,
  version NUMBER DEFAULT 0
);

INSERT INTO account VALUES (1, 10000, 0);
COMMIT;
```

세션 A / 세션 B 두 개의 DB 세션(또는 터미널)을 열고 동시에 실행한다고
가정한다.

------------------------------------------------------------------------

## 1) Dirty Read (오라클에서는 발생하지 않음)

### 세션 A

``` sql
UPDATE account SET balance = 0 WHERE id = 1;
-- COMMIT 하지 않음
```

### 세션 B

``` sql
SELECT balance FROM account WHERE id = 1;
```

결과: **10000 조회됨 (0이 아님)**\
→ 오라클은 Undo 스냅샷을 읽기 때문에 커밋 안된 값은 절대 보이지 않는다.

------------------------------------------------------------------------

## 2) Non-Repeatable Read

### 세션 A

``` sql
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;

SELECT balance FROM account WHERE id = 1; -- 10000
```

### 세션 B

``` sql
UPDATE account SET balance = 5000 WHERE id = 1;
COMMIT;
```

### 세션 A

``` sql
SELECT balance FROM account WHERE id = 1; -- 5000 (값 바뀜)
```

→ 같은 트랜잭션인데 값이 바뀜 = Non‑Repeatable Read

------------------------------------------------------------------------

## 3) Phantom Read

테이블:

``` sql
CREATE TABLE reservation (
  id NUMBER,
  rdate DATE
);
```

### 세션 A

``` sql
SELECT COUNT(*) FROM reservation WHERE rdate = DATE '2026-03-01';
-- 결과 10
```

### 세션 B

``` sql
INSERT INTO reservation VALUES (11, DATE '2026-03-01');
COMMIT;
```

### 세션 A

``` sql
SELECT COUNT(*) FROM reservation WHERE rdate = DATE '2026-03-01';
-- 결과 11 (행 증가)
```

→ 팬텀 리드 발생

------------------------------------------------------------------------

## 4) Lost Update

초기값: balance = 10000

### 세션 A

``` sql
SELECT balance FROM account WHERE id=1; -- 10000
```

### 세션 B

``` sql
SELECT balance FROM account WHERE id=1; -- 10000
```

### 세션 A

``` sql
UPDATE account SET balance = 9000 WHERE id=1;
COMMIT;
```

### 세션 B

``` sql
UPDATE account SET balance = 9000 WHERE id=1;
COMMIT;
```

최종 결과: 9000\
정상 결과: 8000\
→ 갱신 손실 발생

### 해결 (비관적 락)

``` sql
SELECT balance FROM account WHERE id=1 FOR UPDATE;
UPDATE account SET balance = balance - 1000 WHERE id=1;
COMMIT;
```

------------------------------------------------------------------------

## 5) Write Skew

의사 당직 테이블

``` sql
CREATE TABLE doctor (
  id NUMBER PRIMARY KEY,
  oncall CHAR(1)
);

INSERT INTO doctor VALUES (1,'Y');
INSERT INTO doctor VALUES (2,'Y');
COMMIT;
```

### 세션 A

``` sql
SELECT * FROM doctor WHERE oncall='Y';
-- 2명 확인
UPDATE doctor SET oncall='N' WHERE id=1;
-- 아직 COMMIT 안함
```

### 세션 B

``` sql
SELECT * FROM doctor WHERE oncall='Y';
-- 여전히 2명으로 보임
UPDATE doctor SET oncall='N' WHERE id=2;
COMMIT;
```

### 세션 A

``` sql
COMMIT;
```

결과: 당직 의사 0명\
→ 비즈니스 규칙 붕괴

### 해결

``` sql
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
```

------------------------------------------------------------------------

## 6) Dirty Write (DB가 자동 차단)

### 세션 A

``` sql
UPDATE account SET balance = 0 WHERE id=1;
-- COMMIT 안함
```

### 세션 B

``` sql
UPDATE account SET balance = 5000 WHERE id=1;
-- 여기서 대기(wait) 발생
```

→ 세션 A가 COMMIT/ROLLBACK 할 때까지 수정 불가

------------------------------------------------------------------------

## SERIALIZABLE 충돌 감지

### 세션 A

``` sql
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
SELECT balance FROM account WHERE id=1;
```

### 세션 B

``` sql
UPDATE account SET balance=2000 WHERE id=1;
COMMIT;
```

### 세션 A

``` sql
UPDATE account SET balance=1500 WHERE id=1;
COMMIT;
```

결과:

    ORA-08177: can't serialize access for this transaction

→ 오라클은 락 대신 **충돌 감지 방식**으로 직렬성을 보장한다.

------------------------------------------------------------------------

## 낙관적 락 (Version 컬럼)

``` sql
SELECT id, balance, version FROM account WHERE id=1;
-- 예: version = 3
```

수정 시:

``` sql
UPDATE account
SET balance = balance - 1000,
    version = version + 1
WHERE id = 1
  AND version = 3;
```

영향받은 행 수 = 0 이면\
→ 다른 사용자가 먼저 수정한 것 → 재조회 후 재시도

------------------------------------------------------------------------

## 핵심 정리

-   격리수준: 읽기 일관성 제어
-   FOR UPDATE: 수정 경쟁 차단(비관락)
-   version 컬럼: 충돌 감지(낙관락)

**Isolation Level = 무엇을 읽을 수 있는가**\
**Lock / Version = 무엇을 쓸 수 있는가**
