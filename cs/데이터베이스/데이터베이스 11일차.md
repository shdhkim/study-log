# 격리수준별 동시성 이상현상 정리 (Oracle + 일반 DB 비교) 

## 1. 동시성 이상현상 정의

  이상현상              의미
  --------------------- -----------------------------------
  Dirty Read            커밋되지 않은 데이터를 읽음
  Non-Repeatable Read   같은 행을 다시 읽었는데 값이 바뀜
  Phantom Read          조회 행 개수가 바뀜
  Lost Update           수정이 서로 덮어써짐
  Write Skew            논리 규칙(비즈니스 제약) 깨짐
  Dirty Write           커밋 안된 데이터 위에 수정

------------------------------------------------------------------------

## 2. Oracle 기준

### READ COMMITTED (기본)

  이상현상              발생 여부   이유
  --------------------- ----------- -------------------------
  Dirty Read            X           Undo 스냅샷 읽기
  Non-Repeatable Read   O           SELECT마다 새 스냅샷
  Phantom Read          O           새 행 커밋 시 조회 가능
  Lost Update           O           SELECT 시 락 없음
  Write Skew            O           행 단위 보호만 존재
  Dirty Write           X           행 배타락으로 차단

### SERIALIZABLE

  이상현상              발생 여부   이유
  --------------------- ----------- -------------------
  Dirty Read            X           스냅샷
  Non-Repeatable Read   X           동일 SCN 유지
  Phantom Read          X           스냅샷 고정
  Lost Update           X           충돌 감지 후 실패
  Write Skew            X           직렬화 보장
  Dirty Write           X           행락

------------------------------------------------------------------------

## 3. 일반 DB (ANSI/MySQL 등)

### READ UNCOMMITTED

  이상현상              발생
  --------------------- ------
  Dirty Read            O
  Non-Repeatable Read   O
  Phantom Read          O
  Lost Update           O
  Write Skew            O
  Dirty Write           X

### READ COMMITTED

  이상현상              발생
  --------------------- ------
  Dirty Read            X
  Non-Repeatable Read   O
  Phantom Read          O
  Lost Update           O
  Write Skew            O
  Dirty Write           X

### REPEATABLE READ

  이상현상              발생
  --------------------- ------
  Dirty Read            X
  Non-Repeatable Read   X
  Phantom Read          O
  Lost Update           △
  Write Skew            O
  Dirty Write           X

### SERIALIZABLE

  이상현상              발생
  --------------------- ------
  Dirty Read            X
  Non-Repeatable Read   X
  Phantom Read          X
  Lost Update           X
  Write Skew            X
  Dirty Write           X

------------------------------------------------------------------------

## 5. 해석

-   격리수준은 읽기 일관성을 제어한다.
-   Lost Update는 락/버전 전략으로 해결한다.
