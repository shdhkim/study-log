# 격리수준별 동시성 이상현상 정리 (Oracle + 일반 DB 비교)

> 목적: 각 격리수준에서 어떤 문제가 발생/차단되는지 한눈에 보는 표 정리

------------------------------------------------------------------------

## 1. 동시성 이상현상 정의

  이상현상              의미
  --------------------- -----------------------------------
  Dirty Read            커밋되지 않은 데이터를 읽음
  Non-Repeatable Read   같은 행을 다시 읽었는데 값이 바뀜
  Phantom Read          조회 행 개수가 바뀜
  Lost Update           수정이 서로 덮어써짐
  Write Skew            논리 규칙(비즈니스 제약) 깨짐
  Dirty Write           커밋 안된 데이터 위에 수정

------------------------------------------------------------------------

# 2. Oracle 기준 (핵심)

### Oracle 특징

-   MVCC 기반
-   SELECT는 스냅샷 읽기 (락 없음)
-   UPDATE는 행 배타락(X-lock)
-   실제 사용 격리수준: READ COMMITTED / SERIALIZABLE

이미 기본적으로 차단되는 것: - Dirty Read ❌ - Dirty Write ❌

------------------------------------------------------------------------

## Oracle 격리수준별 발생 여부

### READ COMMITTED (기본값)

  이상현상              발생 여부   이유
  --------------------- ----------- -------------------------
  Dirty Read            X           Undo 스냅샷 읽기
  Non-Repeatable Read   O           SELECT마다 새 스냅샷
  Phantom Read          O           새 행 커밋 시 조회 가능
  Lost Update           O           SELECT 시 락 없음
  Write Skew            O           행 단위 보호만 존재
  Dirty Write           X           행 배타락으로 차단

요약: \> 성능 좋음, 대부분 시스템 사용, 대신 동시성 충돌은
애플리케이션이 처리해야 함

------------------------------------------------------------------------

### SERIALIZABLE

  이상현상              발생 여부   이유
  --------------------- ----------- -------------------
  Dirty Read            X           스냅샷
  Non-Repeatable Read   X           동일 SCN 유지
  Phantom Read          X           스냅샷 고정
  Lost Update           X           충돌 감지 후 실패
  Write Skew            X           직렬화 보장
  Dirty Write           X           행락

특징: - ORA-08177 발생 가능 - 재시도 필요

요약: \> 가장 안전하지만 트랜잭션 실패가 발생한다

------------------------------------------------------------------------

# 3. ANSI 표준 DB / MySQL 등 일반적 기준

(일반 교재, PostgreSQL, MySQL InnoDB 개념 이해용)

## READ UNCOMMITTED

  이상현상              발생
  --------------------- --------------------
  Dirty Read            O
  Non-Repeatable Read   O
  Phantom Read          O
  Lost Update           O
  Write Skew            O
  Dirty Write           X (DB가 항상 차단)

------------------------------------------------------------------------

## READ COMMITTED

  이상현상              발생
  --------------------- ------
  Dirty Read            X
  Non-Repeatable Read   O
  Phantom Read          O
  Lost Update           O
  Write Skew            O
  Dirty Write           X

------------------------------------------------------------------------

## REPEATABLE READ (MySQL 기본)

  이상현상              발생
  --------------------- ---------------------------------
  Dirty Read            X
  Non-Repeatable Read   X
  Phantom Read          O (일부 DB는 Gap Lock으로 차단)
  Lost Update           부분 발생
  Write Skew            O
  Dirty Write           X

------------------------------------------------------------------------

## SERIALIZABLE

  이상현상              발생
  --------------------- ------
  Dirty Read            X
  Non-Repeatable Read   X
  Phantom Read          X
  Lost Update           X
  Write Skew            X
  Dirty Write           X

------------------------------------------------------------------------

# 5. 실무 해석 (매우 중요)

격리수준이 해결하는 문제: → 읽기 일관성

격리수준이 해결 못하는 핵심 문제: → Lost Update

그래서 실무에서는:

  상황                 해결 방법
  -------------------- ----------------------------
  수정 화면            SELECT FOR UPDATE (비관락)
  다중 사용자 시스템   version 컬럼 (낙관락)
  금융 마감            SERIALIZABLE
